# IF 可玩支线长篇扩展执行计划

## Goal

把现有 P0-A 至 P0-E 的节点骨架，扩展为可供后续开发拆 JSON、剧情页和选项页的“可玩长支线”文本体系。每条线不按母本小说全文重写，但必须达到与其替代 / 扩展母本段落相称的功能体量：章节、事件、选择、变量回调、现实代价和毕业结算都要完整。

## Key Changes

- 以 `开发规划_IF可玩支线长篇扩展总表.md` 作为长支线体量总控。
- 以 `开发规格_IF路线Schema与P0节点池.md` 作为字段、变量和节点拆分总控。
- 以 `开发索引_IF分支P0文本总索引.md` 作为入口索引。
- 下一阶段先写 `DEFAULT-4XX` 宿舍修补线，作为所有宿舍关系线的模板。
- 模板稳定后再写周屿线、唐骁线、陆沉线、摆烂线、专注感情线、兼职线、完美线和 5X 硬外流线。

## Task List

1. 复核总控文件
   - 打开 `docs/剧情小说母版_v4.0/开发规划_IF可玩支线长篇扩展总表.md`。
   - 打开 `docs/剧情小说母版_v4.0/开发规格_IF路线Schema与P0节点池.md`。
   - 打开 `docs/剧情小说母版_v4.0/开发索引_IF分支P0文本总索引.md`。
   - 确认当前阶段是长支线开发，不是试玩器、小说正文续写或 JSON 节点草案。

2. 编写模板线
   - 新建 `docs/剧情小说母版_v4.0/开发长线_IF可玩支线DEFAULT-4XX宿舍修补线.md`。
   - 按 10 章级结构写：章节定位、现实任务、关系压力、可拆选择、变量变化、回调、毕业结算。
   - 每章至少 2 个有效选择，每 2-3 个选择至少出现一次回调。
   - 保留修补线不圆满原则：增加当面处理机会，不保证大团圆。

3. 扩展宿舍分歧线
   - 复制模板结构到 `R5-ZHOU`、`R5-TANG`、`R5-LUCHEN`、`R5-LIEFLAT`。
   - 每条线必须有被选择对象的吸引力、代价、误读和毕业温度。
   - 未被选择的室友仍按母版继续生活，只减少与林亦舟交集。

4. 扩展外部方向线
   - 编写 `R5-ROMANCE`，保留沈嘉禾 / 夏知微候选结构，不把晚风设为默认必选。
   - 编写 `R4-WORK`，以生活费、排班、留校、快递站 / 打印店 / 勤工为核心。
   - 编写 `R3-PERFECT`，以活动、主持、材料、公开信用和被工具化为核心。

5. 最后处理硬外流
   - 编写 `R5X-HARD`。
   - 保留唯一硬外流规则：进入后第六 / 第七卷只走 5X 单线。
   - 新圈人物必须有姓名和功能，不能是无名 NPC 堆叠。

6. 更新索引和追踪
   - 每完成一条长线，更新 `开发索引_IF分支P0文本总索引.md`。
   - 更新 `task_plan.md`、`progress.md`、`docs/剧情小说母版_v4.0/progress.md`、`findings.md`。
   - 提交 Git，提交信息说明本批路线名称。
