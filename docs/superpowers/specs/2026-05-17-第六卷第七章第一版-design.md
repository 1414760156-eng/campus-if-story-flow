# 第六卷第七章《第一版》设计记录

## 写作目标

- 承接第六章结尾：周一下午八号教学楼机房检查第一版可点击页面、团队名单、三条测试记录和一页说明。
- 把“第一版能打开”写成真实压力：页面能点，但第二次点击、清除记录、说明文字和测试记录之间出现差异。
- 宿舍四人优先：唐骁负责页面和现场回应，林亦舟负责测试记录和部分文字，周屿只做说明页口径检查，陆沉提供公开规则核对和记录格式，不越过各自边界。
- 让冯助教和陈老师的追问逼出修订顺序，而不是把检查写成一次通过。
- 控制第六卷长度：本章完成第一版检查与修订顺序入口，后续进入“周三 C407 修订顺序表”，不扩写成完整答辩。

## 场景与动线

- 核心场景为八号教学楼机房：机房门口交纸质材料、机房内打开页面、投影或教师电脑查看、现场写测试记录。
- 辅助场景为青枫居 4XX 和阳光书屋短场：检查前夜整理说明页，检查后四人把冯助教的清单带回桌面。
- 不使用明德楼、不使用 A302，不新增校外交通。

## 人物动作

- 林亦舟：不把页面问题写成“基本正常”，而是把“第二次点击复用旧时间”“清除后仍留一条记录”写进纸面，承认问题存在。
- 唐骁：本能想先刷新、先改完再给人看，但在老师追问下说出“这是第一版问题，不是展示效果”，并接下修订顺序。
- 周屿：删掉说明页里“提升效率”“减少纠纷”等大词，把描述压成可被问住也能守住的三句。
- 陆沉：指出公开规则来源和页面模拟数据的区分，提醒记录必须有访问时间和页面名称。
- 晚风：只通过微信短句出现，提醒林亦舟别把页面崩掉写成能用，保持现实联系但不抢宿舍线。

## 章节钩子

- 章末落在冯助教通知：周三下午 C407 带修订顺序表、最新版测试记录和负责人签字；第一行必须写清“先修哪个问题，谁修，为什么先修”。
- 形成下一章入口：修订顺序会把“谁修”和“谁承担解释”重新压回青枫居公共桌。

## 文风约束

- 正文不写系统口吻、变量口吻和上帝视角提示。
- 不使用“让林亦舟意识到”“下一句话会”“你怕我怕他怕”等总结句。
- 用纸张、投影、旧时间、按钮、勾选框、手写备注、窗口风声推进人物心理。
