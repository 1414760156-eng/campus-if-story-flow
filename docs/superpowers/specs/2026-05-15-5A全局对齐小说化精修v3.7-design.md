# 5A 全局对齐小说化精修 v3.7 Design

## 目标

在 v3.6 全局审计和 `chinese-novelist` 重排基础上，精修 5A playable 数据。重点不是新增大量节点，而是让关键承重节点带有全局来源、具体生活冲突、第六幕入口和终章回响。

## 修改范围

- `playtest/5a-full-data.test.js`
- `playtest/5a-full-data.js`
- `playtest/5a-full-playtest.html`
- `docs/5A节点验收表_v2.md`
- `1-6幕剧情优化稿.md`
- `沉浸式剧情互动系统_v2.json`
- `findings.md`
- `progress.md`
- `task_plan.md`

## 设计要求

1. 标题升级为 v3.7，并说明“全局对齐与小说化重排”。
2. 入口必须呈现多线同屏压力：宿舍群、生活费/红包、晚风微信和抖音、共享文档、活动证明。
3. 旧账节点必须从 1-4 幕长出来：手机袋、水费、热水卡、外放、空调、陆沉说“没事”。
4. 摔门风险必须同时支持同质收束和唯一硬外流，不能只有硬外流。
5. 饭桌测试必须写出西园餐厅的公开性：筷子、排队、空椅子、手机弹窗。
6. 期末复发必须写成第六幕入口，不把修补写成当场圆满。
7. 5X 外流必须改为“第六幕与终章仅走外流单线”，不再使用“第七幕”表述。
8. 玩家可见文本继续禁止系统说明、上帝视角和元叙事。

## 验收方式

先在 `playtest/5a-full-data.test.js` 加 v3.7 验收，确认红灯，再改 `playtest/5a-full-data.js` 到绿灯。最后跑 full data、app smoke、engine、语法和 JSON 验证。
