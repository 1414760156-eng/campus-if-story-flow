# 武生院 1-6 幕剧情与互动系统优化设计

## Summary

本次优化基于已有 1-6 幕剧情稿、Codex 接入指引、AI 制图细则与终章总控文档，不重新创作世界观，不改变主线关键事件，不改变宿舍四人关系的基本走向。目标是在保持主线完整的前提下，把支线人物从轻量 NPC 扩展为可被玩家互动、触发、记忆和回收的人物事件系统，同时提升文本的小说化可读性和沉浸式剧情游戏感。

## Locked Mainline

必须保留的主线骨架：

- 序幕与第一幕：录取通知、家庭饭桌、宿舍群、武生院站入校、图书馆西门报到、学院确认、中南公寓 4XX 宿舍、北园第一顿饭、父母离校、第一晚宿舍灯光。
- 第二幕：军训服在图书馆仓库领取，日常训练在晨光体育场，手机袋和物资区事件保留，汇演和彩排在健源体育场，军训只写边缘参与和小责任。
- 第三幕：早八、专业导论、小组作业、晨光大道招新、九龙壁舞台/迎新晚会、周屿主持/串场种子、第一学年收尾。
- 第四幕：暑假不是暂停键，家里现实、光谷活动、晚风现实化、夏知微空校园、沈嘉禾资料、陆沉勤工、返校班会通知。
- 第五幕 / 5C：晚风不再只是夜里出口，而是进入现实时间、作息、责任、边界和关系定义的关系。
- 第六幕：从第五幕后寒假到大四上冬季形成准结局，终章只微调关系温度、毕业镜头和短期落点。

## Character Arc Rules

宿舍四人优先：

- 林亦舟：玩家视角，不能突然成为控场型主角；成长来自在具体小事里学会说明、承认有限、整理生活、修补关系。
- 周屿：轻松、会开玩笑、有表达欲；主持和舞台线必须从玩笑、串场、展示表达慢慢长出来，不能空降成舞台中心。
- 唐骁：规则意识、流程感、现实边界；他的弧光不是冷酷，而是用说清楚保护别人和自己。
- 陆沉：沉默、现实压力、可靠但不擅长求助；玩家互动应体现“不追问也不是不关心”，让他逐步多说一点。

支线人物要从 NPC 升级为事件触发者：

- 沈嘉禾：资料、学业、可靠准备；提供学业/升学/资料整理方向的资源与镜像压力。
- 夏知微：记录、影像、作品；提供观察校园、作品集、回看时间的支线。
- 秦越：活动资源、人情网络、实践机会；提供资源便利和边界风险。
- 晚风：线上关系现实化；提供情绪连接、边界练习和现实责任冲突。
- 父母：家庭现实和城市选择；通过饭桌、电话、体检/工作压力自然呈现。
- 陈老师 / 学工老师：规则、班会、现实提醒；只做引导和校准，不替玩家解决问题。

## Interaction Design

在已有 auto/choice 逻辑上增加一层 sidecar 互动：

- 每个主线节点可附加 0-3 个支线 hook，只在玩家已见过对应人物或场景时显示。
- 支线 hook 可以改变心理暗线、关系温度、资源、疲劳、边界意识、修补能力，但不能改变主线事件是否发生。
- 每个重要选择必须有即时反馈和延迟回响，尤其第六幕必须体现几周后、暑期后、冬季前的结果。
- 玩家可见文本不出现主线、副线、权重、flag、触发、系统变量、Codex 等开发语言。
- 场景必须局限在已有武生院沉浸版场景或终章允许的家庭/城市落点内，并遵循空间动线。

## Data Output

交付两个核心文件：

- `optimized_story_acts1_6.md`：按幕输出优化后的剧情结构、人物状态、心理暗线、宿舍关系、支线事件与示例文本。
- `interactive_system_v2.json`：输出可接入的结构化互动系统，包括角色、状态字段、节点、支线 hooks、选项效果、时间回响、场景限制和终章回收。

## Validation

完成前必须检查：

- JSON 语法有效。
- 主线锁点关键词存在：军训手机袋、汇演、班会、主持招新、毕业活动。
- 宿舍四人姓名存在并保留。
- 支线人物至少包括沈嘉禾、夏知微、秦越、晚风、父母/家庭。
- 互动机制包含心理暗线、关系温度、时间回响、资源奖励或疲劳/边界机制。
- 玩家可见文本规则明确禁止开发层语言。
