# 第四卷第二十九章《序号二十二》设计说明

## 目标

承接第二十八章末尾“提交序号 22 被抽查”，写成一章完整课堂压力场。重点不是让抽查变成一次胜利展示，而是让青枫居四个人都被问到自己负责的部分，形成“不能再由一个人替全组兜底”的现实后果。

## 场景与动线

- 起点：青枫居 4XX 清晨，电脑、U 盘、饭卡、润喉糖、便利贴和晚风常温水照片都留在桌面。
- 中段：西园餐厅短早餐后进入教学楼 A302，教师和助教按提交序号抽查 18-23。
- 主要压力：序号 22 上台，陈老师不让其中一人讲完整组，逐段点问每个成员。
- 收束：下课后回到西园或教学楼走廊，互评表 / 个人说明表打开，下一章压力从“能不能讲”转向“怎么写贡献”。

## 人物分工

- 林亦舟：负责整合、提交和为什么等陆沉回来再点提交。他不能把答案说成主持式总结，要承认压缩包里的每个文件都对应具体的人。
- 周屿：嗓子仍哑，负责异常处理。少说废话，用补充样例 2 说明“不入队”的边界。
- 唐骁：负责代码框架和文件顺序。被陈老师截停一次，不能替陆沉讲输出。
- 陆沉：负责输出说明。疲惫、饭卡和勤工痕迹仍在，但必须自己讲。

## 文风约束

正文只写场景、动作、对话和物件。不写系统口吻，不写解释性心理总结，不出现“玩家可见文风禁令”中的词。课堂压力用投影、鼠标、纸面队列图、嗓子、粉笔灰和同学低声议论呈现。

## 章末钩子

抽查通过不是本章终点。章末让助教打开组内互评 / 个人说明补交要求，提醒他们“能讲清楚”之后还要面对“写清楚贡献”的更细压力。
