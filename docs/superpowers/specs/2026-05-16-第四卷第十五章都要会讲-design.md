# 第四卷第十五章《都要会讲》设计记录

## 使用技能

- superpower-bootstrap：继续遵守技能检查和任务追踪。
- superpower-brainstorming：先审本章事件承重，再进入正文。
- planning-with-files：读取现有计划、发现记录和进度，保证续写不漂移。
- superpower-writing-plans：将本章拆成可执行小计划。
- chinese-novelist：正文坚持展示而非讲述，用动作、场景和对话承载心理。
- superpower-verification：完成后用字数脚本和检索命令做新鲜验证。

## 本章承接

第十四章结尾停在陈老师群消息：第二节课随机抽两组演示，组内成员都要能回答分工细节。第十五章必须把“昨晚刚修好的协作”放进课堂公开压力里检验。

## 本章职责

1. 主场从青枫居早晨移到教学楼课堂，地点必须为普通教学楼，不使用明德楼。
2. 公开演示不写成单纯胜利，要写出通宵后的疲态、U 盘备份、同学旁观、老师追问和昨晚版本事故的余波。
3. 四人都要在课堂上有可见分工：周屿讲流程图，唐骁讲代码框架，陆沉讲测试用例，林亦舟讲整合和提交。
4. 陈老师的功能不是说教，而是通过具体追问把“组长”和“实际贡献”的落差摆到台面上。
5. 课堂结束后用食堂、楼道或阳光书屋短场景收束，让四人不是靠宣言修补，而是通过下一次吃饭、转发文件、少抢一句话完成轻微前进。
6. 结尾留下一条新压力：组内互评、下一次实验或运动会/证书时间冲突，为下一章继续推进第四卷压力网。

## 文风与禁线

- 不使用“让林亦舟意识到”“他突然发现”“下一句话会”等上帝视角。
- 不写“周屿怕/唐骁怕/陆沉怕”并列总结。
- 不写游戏开发词：主线、副线、节点、触发、判定、回主线、资源线。
- 心理通过投影灯、U 盘、粉笔灰、键盘、同学窃笑、饭盘和消息震动呈现。
