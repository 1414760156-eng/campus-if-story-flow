# 5A 回环关键节点设计

## Summary

玩家指出“走廊返寝室”不应强迫进门。这个判断成立：空间动作类选择天然具备路线意义，不能像普通心理选项一样只改状态。5A 完整试玩版需要加入“回环关键节点”规则，让玩家可以短暂离开寝室主轴，但最终仍回到宿舍关系线，不破坏第五幕单线收束。

## Chosen Approach

采用“主轴 + 短支线 + 明确回流点”的结构。

- 主轴节点继续承担宿舍关系推进。
- 空间动作、是否进入现场、是否当面谈、是否离开饭桌这类选择可以进入短支线。
- 短支线只处理一次人物互动、心理暗线或资源变化，不展开成新的完整线。
- 短支线结尾必须回到指定主轴节点。
- 普通回复消息、沉默、语气调整等选择只改变心理或关系状态，不改路线。

## First Implementation Target

先改 `ACT5_5A01_CORRIDOR_RETURN`。

新增一个“不进门”的选择：

- 玩家可以突然厌烦，不进寝室，去寝室楼外长椅坐一会儿。
- 支线可以触发晚风聊天，或碰见出来买夜宵的室友。
- 支线只改变“回避值 / 晚风牵引 / 宿舍互信 / 现实压力”等状态。
- 支线最终回流到 `ACT5_5A02_DOORWAY_PAUSE`，让玩家仍然面对宿舍门口的外卖袋、空调遥控器和公共桌面问题。

## Route Shape

示例结构：

```text
ACT5_5A01_CORRIDOR_RETURN
  ├─ 直接进门 / 先听两秒 -> ACT5_5A02_DOORWAY_PAUSE
  └─ 不进门，去楼外坐一会儿 -> ACT5_5A01A_OUTSIDE_BENCH -> ACT5_5A02_DOORWAY_PAUSE
```

这对应用户说的“关键节点可以最后还是回到寝室”，也就是非线性的 `1 -> 4 -> 1 -> 5 -> 2 -> 6` 式体验：路线绕了一下，但主线仍然收束。

## Acceptance Criteria

- `ACT5_5A01_CORRIDOR_RETURN` 至少有一个明确“不进门”的选项。
- 不进门选项必须解锁短支线节点 `ACT5_5A01A_OUTSIDE_BENCH`。
- 原本进门相关选项必须直接回到 `ACT5_5A02_DOORWAY_PAUSE`，避免所有玩家被迫走支线。
- 短支线节点必须标记为回环节点，并指定回流点 `ACT5_5A02_DOORWAY_PAUSE`。
- 短支线必须保留 80 字以上选择前铺垫和 80 字以上选择后反馈。
- 完整数据仍需满足 20 次以上有效选择、40 个以上总选项和引擎可通关。

## Open Risks

- 不进门支线如果写得太长，会把 5A 宿舍线误导成晚风线。因此本轮只做一次短触发，不展开晚风完整关系线。
- 如果后续多个关键节点都加回环，需要建立统一字段，例如 `回环节点`、`回流节点`、`路线级选择`，避免 JSON 变成无规律跳转。
