# 5A 垂直切片试玩器 Design

## 背景

用户确认下一步先做试玩器。当前 `沉浸式剧情互动系统_v2.json` 已有第五幕入口、分线整理、5A 宿舍修补线节点链和强冲突规则，但它仍是静态数据，无法证明玩家点击后的节奏、反馈和状态变化是否成立。

本轮目标不是做完整 H5，也不是做最终视觉包装，而是做一个本地可打开的灰盒试玩器。它服务于剧情调试：玩家能从第五幕入口进入 5A，点击选项，看到即时反馈、状态变化、历程记录和最终宿舍线输出。

## 方案比较

### 方案 A：继续只改 JSON 和剧情稿

优点是快，能继续扩分线。缺点是无法试玩，无法发现节点节奏、状态反馈、结局触发是否真的好用。这个阶段继续纯写稿会积累体验风险。

### 方案 B：直接做完整 H5 原型

优点是视觉接近最终产品。缺点是范围过大，会把美术、路由、存档、音效和正式 UI 都提前引进来，反而拖慢对 5A 可玩性的验证。

### 方案 C：静态灰盒垂直切片

这是推荐方案。用不依赖安装的 HTML、CSS、JavaScript 做一个本地页面，先跑通 `ACT5_00_CONFUSION_AUDIT`、`ACT5_01_LINE_ROUTER` 和 `ACT5_5A01` 至 `ACT5_5A99`。页面重点显示剧情、选项、状态条、日志和结局输出。

## 范围

试玩器只覆盖：

- `ACT5_00_CONFUSION_AUDIT`
- `ACT5_01_LINE_ROUTER`
- `ACT5_5A01_DORM_COLD_START`
- `ACT5_5A02_OLD_DEBTS_SURFACE`
- `ACT5_5A03_LUCHEN_HALLWAY_LISTEN`
- `ACT5_5A04_RULE_REOPEN_MEETING`
- `ACT5_5A05_AFTER_ARGUMENT_MEAL`
- `ACT5_5A06_EXAM_WEEK_RELAPSE`
- `ACT5_5A99_DORM_OUTPUT`

不覆盖 5B、5C、5D、5E、5F，不做正式存档系统，不接入美术素材，不做账号、不做后端。

## 文件结构

| 文件 | 职责 |
|---|---|
| `playtest/engine.js` | 纯逻辑核心：状态初始化、选项效果应用、节点跳转、结局判定 |
| `playtest/5a-data.js` | 5A 试玩节点数据，来自现有 JSON 的垂直切片 |
| `playtest/app.js` | 浏览器交互层：渲染节点、按钮、状态栏、日志、重开和导出 |
| `playtest/styles.css` | 灰盒界面样式，保证可读、可点、移动端不挤压 |
| `playtest/5a-playtest.html` | 本地入口页面，直接双击或浏览器打开 |
| `playtest/engine.test.js` | Node 测试脚本，验证核心状态逻辑和结局分流 |

## 交互设计

页面分四个区域：

1. 顶部：当前节点标题、分线标签、进度位置。
2. 中央：剧情摘要和选项按钮。按钮显示选项文本，点击后进入反馈。
3. 右侧或下方：状态面板，显示 `宿舍互信`、`回避值`、`规则可信度`、`宿舍火药味`、`公开站队`、`短期决裂`、`低温共处`、`修补温度`。
4. 底部：游玩日志，记录玩家选择、即时反馈和状态变化。

关键交互：

- 点击选项后，状态立即更新，日志追加一条记录。
- 如果选项含有 `解锁`，试玩器优先跳到解锁节点。
- 如果没有 `解锁`，按 5A 节点链顺序推进。
- 在输出页根据当前状态显示最合适结果：宿舍修补、短期决裂、公开站队、低温共处。
- 提供 `重新开始` 和 `导出记录`。

## 结局判定

输出优先级：

1. `短期决裂 >= 1` 或 `短期决裂风险 >= 1`：短期决裂。
2. `公开站队 >= 1`：公开站队。
3. `宿舍互信 >= 60` 且 `规则可信度 >= 5`：宿舍修补。
4. 其他情况：低温共处。

初始状态给一个中性但可变的灰盒值：

```json
{
  "宿舍互信": 45,
  "回避值": 0,
  "规则可信度": 0,
  "宿舍火药味": 0,
  "公开站队": 0,
  "短期决裂": 0,
  "短期决裂风险": 0,
  "低温共处": 0,
  "修补温度": 0,
  "现实压力": 55
}
```

## 测试要求

先写 `engine.test.js`，确认在没有 `engine.js` 时测试失败，再实现核心逻辑。测试覆盖：

- 初始状态正确。
- 状态影响可以累加和减少。
- 带 `解锁` 的选项会跳转到指定节点。
- 无 `解锁` 的选项按节点链前进。
- 输出优先级正确，短期决裂高于公开站队，公开站队高于宿舍修补。
- 完整点击路径能从入口推进到 5A 输出页。

## 验收标准

- `node playtest/engine.test.js` 通过。
- `playtest/5a-playtest.html` 存在，可本地打开。
- 页面包含当前节点、选项按钮、状态面板、日志、重新开始、导出记录。
- JSON 源文件仍能解析。
- 5A 节点链仍为 7 个节点，选项数保持 4、4、4、5、4、4、4。
