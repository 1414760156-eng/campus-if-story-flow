# 第五卷第八十九章《草稿箱》设计

## 使用技能

- planning-with-files：承接第八十八章断点和项目进度。
- brainstorming：本章只追 `发布端草稿_今晚先发` 如何靠近发布，不新开事故。
- writing-plans：写清文件、人物、证据和收束钩子。
- chinese-novelist：通过屏幕、电话、门禁、截图和人物停顿展示冲突。
- verification：写后统计并扫描禁用表达、地点误用和不宜词。

## 本章目标

承接第八十八章末尾：终版候选已经躺进 `发布端草稿_今晚先发`。本章追查它是否进入公众号草稿、谁粘贴、谁预览、谁发给老师看，把“候选”与“快要发布”的距离写实。

## 场景与人物

- 主场仍在东区商业街后台，旧一体机、牛皮纸袋、截图和现场人员继续压着空气。
- 许棠在明德楼融媒体工作室通过电话/截图接入。明德楼只作为融媒体工作场景使用，符合地点规则。
- 梁思予从宿舍楼下等待电脑交接，不让她单独承担全部责任。
- 宿舍四人功能不变：周屿守边界，唐骁追问动作，陆沉落表，林亦舟串联但不离场。

## 章节推进

1. 老师要求查发布端，许棠接电话，说明草稿箱里确有一份未发布草稿。
2. 许棠打开融媒体工作室电脑，按截图确认草稿标题、摘要、正文第一段和保存时间。
3. 草稿正文来自 `正文.txt`，封面摘要来自 `封面摘要.txt`，但许棠只做过格式检查和预览，没有正式发布。
4. 唐骁追问“没发出去”和“已经生成预览”的差别；周屿提醒不能用“未发布”掩盖它已经被看见。
5. 陆沉记录：进入草稿、预览生成、发送预览链接、接收对象、是否外传。
6. 林亦舟收到晚风短消息，但只回复一句“还在查文字”，不让晚风吞掉现场。
7. 章末发现预览链接曾在 02:56 发进“发布前确认”三人小群，而小群里还有老师以外的一个头像：`XIAO-QIN` 或秦越的小号/工作号。为了不把秦越强行推成罪人，钩子只落在“确认群里多了一个人”。

## 风格提醒

- 不写系统、节点、触发、判定等游戏口吻。
- 不用“让他意识到”“下一句话会”这类上帝视角。
- 冲突靠对话和证据推进，不靠作者总结。
- 不让许棠成为唯一责任人；她是发布端动作的证人和参与者。
