# 第四卷第二十七章《三分钟》设计说明

## 目标

承接第二十六章末尾“每个人写自己能讲的三分钟，不互相代写”，把数据结构实验三的责任从时间轴变成青枫居 4XX 的当面排练。章节重点是让四个人都暴露各自不擅长的部分：周屿讲异常处理讲不清，陆沉排班换不掉，唐骁二级复盘后仍想兜底，林亦舟必须回应晚风但不能借“忙”继续拖。

## 叙事策略

- 不直接跳到周日晚提交，先写周六晚上，给周日晚冲突做充分铺垫。
- 场景以青枫居 4XX 为主，辅以白天短场：唐骁二级复盘、陆沉勤工换班消息、周屿放弃东区商业街合乐。
- 三分钟讲解以手机计时、纸面流程、代码输出和桌面物件推动，不写抽象成长。
- 晚风线只作为现实边界：林亦舟回“晚点”以后终于补一段短语音，但不让晚风替他消解宿舍压力。

## 场景安排

1. 周六上午：唐骁在二级群开麦讲第 27 题，讲对了但耗掉精神；林亦舟和陆沉在阳光书屋补实验说明。
2. 周六下午：陆沉在图书馆勤工群问换班失败；周屿从教学楼 B206 回来，带着秦越的走位批注，周六晚合乐缺席成事实。
3. 周六晚上青枫居：四人用手机计时练三分钟。唐骁能讲代码但压不住想替别人修句子；陆沉讲出队条件时卡在变量命名；周屿讲异常处理时把主持稿语气带进来，被唐骁打断；林亦舟讲整合和提交时差点把“我来改”说出口。
4. 晚风语音：林亦舟出门到走廊尽头/水房旁边回一段短语音，说明今晚还要排讲解，没有用“我没事”糊过去。
5. 钩子：周日晚排班仍未换掉，助教又发提交顺序提醒；唐骁在公共桌写下“六点以前只跑流程，不改功能”，但代码里出现一个新样例输出不一致。

## 文风约束

- 使用 chinese-novelist 原则：动作、对话、物件和声音推动。
- 不写系统口吻、路线术语、未来影响说明。
- 避免“觉得、发现、突然、忽然”等惰性表达。
- 不写“让林亦舟意识到”“下一句话会”等上帝视角句。
- 地点限于武生院合理动线：阳光书屋、图书馆、教学楼 B206、青枫居。

## 自检

无占位项；不改变主线锁点；不打开 5X；支线与晚风只做压力交错，不抢宿舍四人主轴。
