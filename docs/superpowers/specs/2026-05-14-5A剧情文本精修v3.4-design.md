# 5A 剧情文本精修 v3.4 设计说明

## 背景

v3.3 已经完成 5A 的玩法厚度：强关系优先、多边缘人物软回流、唯一外流线、微心态聊天节点。下一步不再继续扩机制，而是把玩家实际读到的 5A 文本和选项做小说化精修。

## 目标

- 去掉玩家可见文本里的系统按钮感，例如“确认写入第六幕”“查看关系清单”“系统写入”等。
- 保留玩法结构和状态字段，但把玩家看到的按钮改成角色当下会做的动作。
- 补强宿舍四人的性格弧光：
  - 周屿：用玩笑和面子保护自己，害怕被公开定罪。
  - 唐骁：依赖规则和效率，害怕关系失控。
  - 陆沉：安静但有边界，害怕一开口就被别人代表。
  - 林亦舟：从主持/圆场者转为承认自己也是当事人。
- 输出页也要沉浸化：不再像调试结果页，而像玩家带着一次关系结果离开现场。

## 精修范围

本轮只改 5A 数据文本，不改节点结构：

- `ACT5_FULL_01_ROUTER`：从“系统分线整理页”改为手机消息盘点的沉浸写法。
- `ACT5_5A11_TABLE_TIMER` 至 `ACT5_5A15_DOOR_SLAM_RISK`：强化规则重谈、旧账、站队和摔门风险的对话张力。
- `ACT5_5A20_RELAPSE_CHOICE`：强化期末复发时的真实动作和人物反应。
- `ACT5_5A99_DORM_OUTPUT_FULL`、`ACT5_5A15B_DORM_CONVERGENCE`、`ACT5_5X99_OUTFLOW_OUTPUT`：去系统化，改为可读的沉浸收束。

## 验收标准

- 标题包含 `v3.4`。
- 玩家可见选项和反馈中不得出现 `系统写入`、`导出本次`、`确认本线结果`、`查看被远离的关系清单`、`确认5A同质收束` 等调试式文案。
- 输出页选项必须是沉浸动作，例如夹进文件夹、保留照片、把手机扣下、把拖鞋摆正。
- 关键冲突节点必须包含四人明确反应，不能只写状态变化。
- 现有节点数、选项数、回流/外流结构保持不降级。
