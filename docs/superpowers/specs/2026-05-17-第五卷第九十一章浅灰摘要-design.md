# 第五卷第九十一章《浅灰摘要》设计稿

## 目标

承接第九十章末尾 `改后预览_0306.png`：标题和正文已经改过，预览卡片里的浅灰摘要仍残留旧句。第九十一章不把问题写成单人失误，而是继续拆开“显眼处改了、角落没看”的传播过程，让宿舍四人保持在现场中心。

## 场景

- 主场：东区商业街后台，旧一体机、桌面水痕、白卡、导出聊天记录。
- 远端：明德楼融媒体工作室，许棠在电话里调出公众号草稿、预览截图、摘要编辑记录。
- 不写课程、班会、机房或考试进入明德楼。

## 人物职责

- 林亦舟：只短回晚风，不让外部消息接管现场；继续在桌边看见“看见”和“确认”的区别。
- 周屿：压住把许棠或梁思予单独推出去的冲动，逼大家把文件动作讲清。
- 唐骁：追问“改了哪里”“谁重新看过”“浅灰字算不算发布前检查”。
- 陆沉：把标题、正文、摘要、预览卡片、截图时间和查看者分栏记录。
- 许棠：承认自己检查标题、封面和正文，没有把摘要重新点开看。
- 梁思予：承认自己改过摘要文字，但只发了文本，没有确认它被复制进草稿。
- 秦越：承认自己只看了标题和正文，错过浅灰小字。
- 老师：把“改文字”和“看预览”分开，不替任何人收尾。

## 章节推进

1. 从浅灰摘要被放大开始，后台短暂安静。
2. 核对 03:03 至 03:06 之间的修改：标题、正文、封面、摘要分别来自哪里。
3. 梁思予说明她改过摘要，但用文本发给许棠；许棠在草稿里先改标题和正文，预览图生成后没有重新检查浅灰摘要。
4. 秦越和胡嘉宁都看过预览图，却只看显眼位置。
5. 陆沉把“已改文本”和“已看预览”分开，避免“改过了”遮掉“没看见”。
6. 章末钩子：`改后预览_0306.png` 在许棠桌面被另存为 `发布预览_可用.png`，下一章可追“可用”怎样把停用意见再次冲淡。

## 文风约束

- 不写策划词、变量词、系统口吻。
- 不写“让林亦舟意识到”“下一句话会”等上帝视角。
- 心理靠手、屏幕、浅灰字、桌面水痕、白卡和沉默呈现。
- 结尾留钩子，不做总结。
