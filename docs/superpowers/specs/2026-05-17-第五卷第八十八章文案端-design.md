# 第五卷第八十八章《文案端》设计

## 使用技能

- planning-with-files：沿用母版进度、断点、扫描记录。
- brainstorming：先限定本章只追 `WENAN-LAPTOP`，不新开事故。
- writing-plans：写前明确场景、冲突、证据递进和收束钩子。
- chinese-novelist：展示而非讲述，用动作、屏幕、语音和人物反应推动。
- verification：写后统计字数并扫描禁用表达、地点误用和不宜词。

## 本章目标

承接第八十七章末尾 `导语候选_终版候选.docx` 创建者为 `WENAN-LAPTOP`。本章追清这台文案端旧电脑如何把第二版导语变成“终版候选”，并把压力推向“候选已经进入发布端草稿”的下一步。

## 不动规则

- 宿舍四人仍是观察和推动核心：林亦舟串联现场与外部消息，周屿守边界，唐骁追问动作差别，陆沉落表。
- 不引入 5X 外流，不让晚风线占据核心。
- 不使用明德楼承载课堂、班会、考试等场景；本章仍在东区商业街后台。
- 不把某个同学写成唯一罪人；问题落在公用电脑、临时权限、文件命名和转述链条。

## 章节推进

1. 旧一体机旁，胡嘉宁确认 `WENAN-LAPTOP` 是图文文案组旧笔记本，今晚不在后台，而在前场水牌桌附近用过。
2. 梁思予通过电话/语音说明电脑使用情况：旧笔记本不设严格个人账号，桌面有 `今晚先发`、`导语候选`、`发布端草稿` 等文件夹。
3. 陈景承认自己把第二版发进文案临时群后，有人用旧笔记本下载并整理；胡嘉宁确认自己曾要求“再顺一下，别直接发”。
4. 唐骁追问“顺一下”和“终版候选”的距离；周屿要求先查动作，不用语气替任何人定罪。
5. 陆沉记录：第二版来源、旧笔记本下载、另存为终版候选、文件夹去向、发布端草稿路径。
6. 林亦舟收到晚风“别回太晚”的短消息，按掉屏幕，把注意力留在现场。
7. 章末查到 `终版候选` 的相关位置不只是候选文件夹，而是已经被放进 `发布端草稿_今晚先发`，为下一章追发布端导入做钩子。

## 风格提醒

- 不写系统口吻，不写“让他意识到”等自我总结。
- 不用全知预告句。
- 写具体动作：卷闸门余响、旧电脑屏幕反光、免提里的电流声、表格行列、文件夹名、人物停顿。
