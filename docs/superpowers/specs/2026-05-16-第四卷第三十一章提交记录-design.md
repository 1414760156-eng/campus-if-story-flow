# 第四卷第三十一章《提交记录》设计说明

## 使用技能

- `superpower-brainstorming`：先把本章事件、人物压力和空间动线对齐。
- `chinese-novelist`：正文采用展示而非讲述，靠物件、动作、对话和场景承载心理。
- `planning-with-files`：将本轮设计、计划、进度和校验结果写回文件。
- `superpower-writing-plans`：在动笔前固定执行顺序和验收方式。

## 本章承接

上一章结束在实验四通知：小组代码仓库要登记，后续查看个人提交记录和每周日志，不能再由组长最后统一上传。四份个人说明刚把各自做过和没做过的边界写出来，新的协作压力立刻变成可追溯的时间戳。

## 本章目标

写出“提交记录可见”带来的新压力。它不是新的大灾难，而是把青枫居 4XX 的旧问题换成长期留痕的形式：谁先上传、谁只改了文件名、谁没有自己的账号、谁因为主持、勤工和二级错题错过约定时间，都会被仓库记录拉到桌面上。

## 场景与动线

1. 青枫居 4XX 深夜：四人看完实验四通知，先不争吵，只被“每个人都要有记录”压住。
2. 次日阳光书屋：林亦舟尝试登记仓库，唐骁整理命名和权限，陆沉处理旧电脑与勤工排班，周屿夹在 B206 排练和仓库实名之间。
3. 八号教学楼机房：助教现场提醒不要由组长代传，第一次有效提交要写清楚改动内容。
4. 青枫居 4XX 夜里：四人围着同一张公共桌做第一次提交，唐骁想一次性把框架搭好，陆沉坚持自己上传日志，周屿把提交说明写得太像主持串词，林亦舟把自己的组长权限从“替大家做完”改成“给大家留出口”。

## 人物推进

- 林亦舟：从压缩包整合者转为仓库登记人，学会不把成员痕迹折成自己的总文件。
- 周屿：主持排练仍在拉扯，他第一次把“没来得及”写进技术文件旁边的时间记录里。
- 唐骁：规则感继续提供秩序，但他必须接受规则不能替别人完成责任。
- 陆沉：旧电脑、勤工时间和自尊同时存在，他坚持用自己的账号留下第一条提交。

## 文风约束

正文不写系统提示，不写上帝视角预告，不写抽象成长总结。心理变化通过屏幕、键盘声、权限列表、提交说明、饭卡、旧电脑、排班纸、B206 门缝和阳光书屋桌面的插座来呈现。

## 结尾钩子

第一次仓库登记完成后，教师群追加要求：明晚课堂随机打开仓库历史，抽查每个人第一条提交说明和本周日志。压力从“能不能传上去”转向“传上去的东西是不是本人留下的”。
