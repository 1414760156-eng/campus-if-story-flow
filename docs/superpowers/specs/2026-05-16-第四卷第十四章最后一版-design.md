# 第四卷第十四章《最后一版》设计记录

## 使用技能

- superpower-bootstrap：确认本轮继续沿用技能工作流。
- superpower-brainstorming：先审事件功能、人物压力和悬念位置，再写正文。
- planning-with-files：读取当前任务计划、发现记录和进度，保证接续不漂移。
- superpower-writing-plans：把本章目标拆成可执行检查项。
- chinese-novelist：正文按中文小说口吻写，展示而非讲述，不写系统/游戏提示。
- superpower-verification：写完后用检索和字数脚本复核禁用表达、地点逻辑和累计字数。

## 本章承接

第十三章结尾停在“收到两份同名文件，组长确认一下最终版”。第十四章必须直接接住这个事故，不另开新场景，不让读者空等。

## 本章职责

1. 把“同名压缩包”写成一次真实的宿舍协作旧账：不是谁恶意背刺，而是临近截止时，各自用自己的方式想把事情兜住，结果互相踩到边界。
2. 让林亦舟承担组长责任，但不写成完美调停；他要先处理文件，再处理人。
3. 让周屿的面子、唐骁的规则感、陆沉的低声提醒都落在具体动作上：手机私聊、命名规则、截图、流程图、U 盘、学习委员消息。
4. 保持第四卷引导式外流只作轻影：便利店许澈、夜色、短暂想离开可以出现，但不能让许澈进入关系核心。
5. 保留晚风现实化陪伴：她只能通过消息和语音在场，不替宿舍做判断。
6. 结尾钩子落到第二天随机演示和分工追问，推动下一章进入课堂公开压力。

## 场景与地点规则

- 主场：青枫居 4XX 寝室、楼下外卖架、宿管阿姨窗口。
- 可短暂出现：校园快递站旁路灯、便利店玻璃门、教学楼提醒消息。
- 不使用明德楼承载班会、上课、答疑、机房和补考。
- 不写公交动线；若涉及出校，只保留地铁/步行现实口径。

## 文风禁线

- 不写“让林亦舟意识到”“他突然发现”“下一句话会把今晚推向”等上帝视角句。
- 不写“周屿怕、唐骁怕、陆沉怕”的并列总结。
- 不写“系统、节点、触发、判定、回主线、资源线”等游戏开发口吻。
- 不用长段心理抽象解释，用物件、动作、停顿、消息时间、文件名和声音承载情绪。
