# 第三卷第二十四章《跑不出来》设计记录

## 目标

承接第二十三章末尾程序设计基础第一次上机检查通知，把第三卷从公开表达复盘转入计科学业、主持队、新闻中心与宿舍疲劳的同日挤压。章节要让“边界”从照片材料自然转到代码作业：不代写、不转交完整答案，只讲思路和证据。

## 必守约束

- 课程与上机发生在八号教学楼/计算机相关机房，不使用明德楼承载课堂。
- 明德楼如出现，只能以许棠或新闻中心材料的远程消息存在，不写成课堂或试训场景。
- 宿舍四人同场推进：林亦舟被代码、晚风消息和宿舍节奏拉扯；周屿在 B206 主持队二轮与上机截止之间卡住；唐骁坚持个人代码与提交规则；陆沉用输入输出样例拆解问题，但不代写。
- 支线人物以“轻而有效”的方式出现：罗泽、丁祺、何书南可在机房/群聊里延续 A101 空表；许棠通过材料说明远程进入；秦越通过 B206 通知给周屿压力；晚风通过微信/游戏现实入口轻触林亦舟。

## 文学处理

- 使用 chinese-novelist 原则：用机房风扇声、代码报错、共享文档、提交入口、B206 消息和键盘声推进，不写心理总结。
- 不写游戏系统口吻，不写“变量/节点/判定/触发/回流”等正文禁用词。
- 冲突必须现实：有人想要完整代码，有人只肯讲思路；有人想先去 B206，有人盯着截止时间；有人把公共问题空表挪到寝室充电线，又被指出不能把同学写成对象。

## 章节结构

1. A101 后四人去西园/阳光书屋短暂停留，群里空表和上机题同时刷屏。
2. 八号教学楼机房，第一次上机检查前的代码混乱：输入读不完整、最多字符位置输出不对、截图要求含运行结果。
3. 罗泽来要“参考代码”，唐骁拒绝直接发，陆沉改用样例和流程图讲思路，形成“代码边界”的现实冲突。
4. 周屿收到秦越 B206 二轮提醒，主持稿和实验题撞在一起；林亦舟被晚风问今晚是否上线，他只能把输入框删掉重打。
5. 回青枫居后，四人各自补代码、改材料、压声音，章末以“B206 二轮”和“上机截止”两条消息上下贴着作为钩子。

## 本章钩子

结尾让周屿同时收到秦越的二轮提醒和助教的提交提醒，下一章可继续写周末夜晚的分流：B206、机房、青枫居和晚风消息并行推进。
