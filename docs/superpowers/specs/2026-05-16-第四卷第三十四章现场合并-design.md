# 第四卷第三十四章《现场合并》设计说明

## 写作目标

承接第三十三章结尾的课程群通知，让青枫居四人在八号教学楼机房完成第一次正式分支合并。重点不写“成长总结”，而写四个人在屏幕、冲突文件、审核表和抽问中重新确认协作边界：审核人能提出问题，但不能替别人改完再补一句说明。

## 场景边界

- 可用场景：青枫居 4XX、西园餐厅、八号教学楼机房、教学楼走廊、东区商业街消息回声。
- 不使用明德楼承载课程、班会、机房、答疑或上机。
- 不新增校外交通动线。
- 许澈不出现。

## 人物推进

- 林亦舟：负责把四份审核说明和仓库合并顺序放在一起，但不替每个人回答。
- 周屿：带着东区商业街小舞台开场被黎旻顶上的余波进入机房，在被抽问时把“说一声不是审批权”讲出来。
- 唐骁：二级回放未补完，急躁会落在鼠标和合并按钮上；他要学会把“规范建议”留在审核意见里。
- 陆沉：延续补门禁后的本人记录原则，坚持谁的日志谁解释，谁的代码谁确认。

## 事件设计

1. 青枫居清晨：四人未睡足，唐骁二级回放仍卡在第 18 题，周屿看秦越发来的收尾走位，陆沉把补签纸和旧电脑一起装包。
2. 西园餐厅：用早饭、打印声、准考证和运动会提醒写现实压力，不解释心理。
3. 八号教学楼机房：助教说明现场合并规则，每组审核人要能说清核对内容。
4. 第一次合并：周屿异常输入说明由唐骁审核。唐骁想直接把口语表述改成规范表达，最后写成审核建议。
5. README 冲突：唐骁 README 与林亦舟目录说明在同一行发生冲突，四人围绕“先在四人群说一声”和“相关成员确认”处理取舍。
6. 随机抽问：助教抽到周屿说明唐骁 README，或者抽到唐骁说明周屿异常输入；问题落在“核了什么、哪些没有替改”。
7. 章末钩子：现场合并过关后，课程群要求周五提交合并冲突处理记录；唐骁二级群同时提醒今晚模拟卷关闭，周屿收到秦越收尾名单，陆沉收到图书馆换班消息。

## 文风约束

- 展示而非讲述，不写“让林亦舟意识到”等总结。
- 避免第四卷高风险词：觉得、发现、突然、忽然、下一句话、怕被、怕规则、怕自己。
- 章节结尾保留下一轮压力，不写成圆满和解。
