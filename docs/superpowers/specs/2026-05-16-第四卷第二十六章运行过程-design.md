# 第四卷第二十六章《运行过程》设计说明

## 目标

承接第二十五章末尾的数据结构实验三通知，把公开材料压力转入真实专业课压力。章节要让“组内成员都要能讲”变成四个人周末时间表上的具体麻烦，而不是一句新的口号。

## 叙事策略

- 课堂短场收束空白表：陈老师只用空表讲五分钟，不展示四人材料。
- 实验三落到具体任务：队列模拟与运行过程抽查，要求每个人能解释输入、入队、出队、异常和输出。
- 周末时间冲突并排出现：唐骁二级第 27 题复盘，周屿教学楼 B206 小舞台走位，陆沉周日晚勤工排班，林亦舟负责整合但不能再把所有空缺揽走。
- 场景从青枫居到教学楼 A302，再到八号教学楼机房和阳光书屋，保持武生院动线。

## 场景安排

1. 次日早晨青枫居：周屿嗓子哑、唐骁补第 27 题、陆沉打印排班、林亦舟没有点开晚风深夜视频。
2. 教学楼 A302：陈老师用空白表讲边界，随后发布实验三讲解要求。
3. 八号教学楼机房：四人初试选题，代码能跑但运行过程讲不顺，唐骁不愿只做“救火的人”。
4. 阳光书屋：排周末时间表，陆沉周日晚排班卡住提交前半小时，周屿 B206 排练和唐骁二级冲突，林亦舟接下整合但被要求把可讲部分分给每个人。
5. 结尾钩子：助教在课程群补发抽查名单范围，第一批包含他们组所在的区间；晚风那条视频仍未点开并转成新的语音。

## 文风约束

- 展示而非讲述，用动作、群消息、机房键盘声和表格呈现压力。
- 不写系统口吻，不写未来影响说明。
- 避免“觉得、发现、突然、忽然”等惰性表达。
- 不写“下一句话会”“让林亦舟意识到”等上帝视角句。
- 不使用玩家路线术语。

## 自检

无占位项；不改变主线锁点；不提前开启 5X；章节收束公共材料并连接专业课压力。
