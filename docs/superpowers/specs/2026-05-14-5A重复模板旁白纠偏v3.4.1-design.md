# 5A 重复模板旁白纠偏 v3.4.1 设计说明

## 背景

试玩中出现了反复重复的反馈句：

`这一步会被记进……等下一次同一张桌子、同一扇门、同一条群消息再次出现时，语气会悄悄变重或变轻。`

这句话来自自动补字数逻辑，不是具体节点剧情。它会让玩家感到每个选择都被同一套系统提示解释，削弱沉浸感。

## 目标

- 删除玩家可见文本中的重复模板句。
- 保留选择反馈不少于 80 字的可玩性密度要求。
- 数值变化继续写在 `状态影响`，但不在剧情反馈里直接解释变量记账。
- 用场景余波替代模板说明：杯盖、笔尖、门缝、床帘、手机屏幕、便签纸、相机带、便利店玻璃等物件承接心理变化。

## 修改范围

- `playtest/5a-full-data.test.js`：新增禁止重复模板旁白的测试。
- `playtest/5a-full-data.js`：改写 `enrichFeedback()` 的短反馈补足逻辑。
- `task_plan.md`、`findings.md`、`progress.md`：记录本次纠偏。

## 验收标准

- `node playtest/5a-full-data.test.js` 通过。
- 玩家可见文本不得出现：
  - `这一步会被记进`
  - `同一张桌子、同一扇门、同一条群消息`
  - `语气会悄悄变重或变轻`
- `node --check playtest/5a-full-data.js` 通过。
- 5A 节点数、选项数、回流/外流结构不变。
