# 第四卷第三十章《写不到的地方》设计说明

## 目标

承接第二十九章的课堂抽查通过与助教要求补交个人说明。上一章解决的是“每个人能不能亲口讲清楚”，本章推进到“每个人能不能在纸面上只写自己真正做过的事”。它要把宿舍四人的互信继续压细，不重复课堂展示，也不把材料补交写成机械流程。

## 场景与动线

- 开场：A302 下课后到西园餐厅，四个人看模板字段，讨论“未参与部分”怎么写。
- 中段一：周屿去教学楼 B206 听秦越十分钟开头，避免用嗓；陆沉去图书馆临时盘点期刊架；唐骁和林亦舟回青枫居先写模板。
- 中段二：青枫居 4XX 夜里四张桌子分别亮屏，四个人把自己做过的、没做过的和不能替别人写的地方落成文字。
- 章末：个人说明提交前，课程群出现下一项课程设计预告 / 小组仓库登记要求，提醒专业课压力没有停下来。

## 人物压力

- 林亦舟：作为组长容易把全组整合写成自己的贡献，本章要让他把“等陆沉回来确认”写成过程，而不是功劳。
- 唐骁：不喜欢空白栏，倾向把流程补齐。本章要让他面对“没做就留空”的不适。
- 周屿：主持队、B206 和嗓子压力并行。他要把异常边界写成具体修改，而不是写成好听的表达。
- 陆沉：图书馆勤工占用时间，但输出说明必须由他自己写。他不能让手机备忘录替自己完成，也不能让别人帮他补。

## 文风约束

遵守 `chinese-novelist` 原则，展示而非讲述。个人说明不能写成表格说明文，要通过屏幕光、光标、删除键、背包、饭卡、润喉糖、键盘声和互相看文件的动作表现关系变化。正文禁用系统/游戏词，不写解释性心理总结。

## 章末钩子

个人说明提交成功后，不写彻底松口气。章末让课程群弹出“实验四需要创建小组仓库，提交 commit 记录和个人日志”的预告，下一章可自然进入代码仓库、贡献记录和更长周期协作压力。
