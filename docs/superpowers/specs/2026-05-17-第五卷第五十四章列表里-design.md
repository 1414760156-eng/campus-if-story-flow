# 第五卷第五十四章《列表里》设计记录

## 目标

承接第五十三章末尾“旧字幕条已经提前导入播放软件”。本章处理“删掉、停用、留档”的现实差别：旧字幕不能留在播放列表里，也不能无记录地直接消失。

## 约束

- 不开启 5X，不引入许澈。
- 场景仍在东区商业街小舞台、侧屏控制台、台侧和活动群。
- 宿舍四人继续分工：唐骁确认播放列表与字幕截图，陆沉留存停用前后记录，周屿确认旧字幕不再包装自己，林亦舟负责在台侧与控制台之间传话。
- 融媒体老师只给现场工作标准，不代替主角团解决。
- 叶青作为已出现的剪辑/字幕志愿者承担控制台执行，不新增陌生核心人物。

## 事件链

1. 听到旧字幕条仍在播放软件列表里，唐骁要求先停止试屏，不让旧条进入屏幕。
2. 众人从小舞台台侧转到侧屏控制台，看到播放软件里有“旧摘要”“新同源稿”两个字幕条。
3. 秦越想直接删旧条，陆沉要求先截图记录它已经停用，避免事后说不清。
4. 融媒体老师区分“播放列表删除”和“素材留档”：列表里不能有旧条，留档文件可以放到停用文件夹。
5. 叶青按要求复制旧条到“停用勿播”文件夹，再从播放列表移除。
6. 唐骁核对新条的三行文字、起止时间和自动播放顺序。
7. 夏知微提醒控制台屏幕也可能被花絮扫到，旧条窗口不能停留在桌面。
8. 旧条从播放列表消失，新条顺序正确，但侧屏试黑底时短暂闪出旧句残影。

## 写法

用控制台桌面的数据线、播放软件列表、黑底预览窗、鼠标点击、截图回传、舞台侧屏闪动等具体动作推进。避免抽象说教，让“留证”和“防止误播”之间的拉扯靠操作顺序体现。
