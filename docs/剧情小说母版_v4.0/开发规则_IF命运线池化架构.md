# IF 命运线池化架构

## 用途

本文用于约束后续所有 IF 可玩支线开发，解决“所有支线都往同一条主线里挤，导致节点、变量和人物关系互相打架”的问题。

核心判断：本项目只有一条恒定不动的真实世界线，即小说母本。玩家一旦在关键时期做出足以改变林亦舟生活重心的选择，就不再把其它支线继续塞进同一条主线，而是进入一个新的命运池。后续主线锁点仍会发生，但以该命运池的视角重新嫁接、重新结算。

2026-05-19 追加明确：后续所谓“支线”统一视为**命运外流线**。它不是主线旁边的小外传，而是玩家在关键抉择点引发蝴蝶效应后，林亦舟后续大学生活进入的新池子。第一卷和第二卷作为公共序章共享；第三幕以后只在关键抉择点进入外流线，进入后各走各的。

## 六层结构

| 层级 | 名称 | 作用 | 开发约束 |
|---|---|---|---|
| L0 | 小说母本线 | 唯一恒定世界线，负责人物原始性格、事件顺序、校园时间和生活半径 | 不被玩家改写 |
| L1 | 共享序章 | 第一卷、第二卷共享，完成入校、4XX 初识、军训手机事件等基础关系 | 不开长线外流，只积累初始变量 |
| L2 | 命运分歧入口 | 第三幕、第四幕、第五幕的关键选择点 | 只在少数关键抉择开放，不在每个事件开树 |
| L3 | 独立命运池 | 玩家进入某条 IF 后，该线拥有自己的事件链、变量池和回流点 | 不再和其它命运线交叉开新主分支 |
| L4 | 公共锁点嫁接 | 汇演、班会、主持招新、站队、第六卷关系定形、第七卷毕业流程等仍发生 | 每条线写自己的版本，不取消锁点 |
| L5 | 毕业结算 | 按该命运池内长期投入的人、生活方式和代价生成结局温度 | 不用其它线变量抢结局 |

## 基本规则

1. **母本线恒定**
   小说母本是默认世界真实线。周屿、唐骁、陆沉、晚风、夏知微、沈嘉禾、秦越、许棠、许澈和生活 NPC 都按母本生活半径继续推进。玩家不选择他们时，他们不会消失，也不会为林亦舟停在原地。

2. **第一二幕共享**
   第一卷和第二卷是公共序章，负责建立 4XX 基础关系、军训手机事件、晚风初连线、规则感、公开边界和回避倾向。这里可以有微选择，但不直接开长线命运池。所有长线外流从第三幕开始。

3. **关键抉择即外流**
   第三幕不是“选不选融媒体”或“直接进入完美线”，而是社团、主持招新、活动圈、摄影、新闻中心、幕后物资、公开表达和回寝室之间第一次真正分发林亦舟的生活重心。默认主轴本身不再无限外流：从宿舍主轴能直接进入的完整关系线只有宿舍站队线和晚风线。其它感情线必须先在社团 / 活动父池里出现明确女生角色和连续生活半径，不能从主轴凭空接出。

4. **父池先分发，子外流再展开**
   第三幕这类入口必须先进入父池，再在父池内部按持续选择生成子外流。父池负责判断“林亦舟此后主要在哪种生活半径里活动”，子外流负责展开该半径里的主事件、内回流和毕业结算。融媒体、主持队、摄影社、幕后物资都不能单独抢成第三幕根路线，它们只是 `POOL-A3-ACTIVITY-PUBLIC` 下的不同 focus。

5. **命运线互斥**
   玩家进入一条可玩命运线后，其它命运线不再作为完整可选主分支开放。其它线人物可以出现，但只能是软照面、群消息、朋友圈、毕业流程同场或只读回声。

6. **公共事件版本化**
   同一个公共锁点可以被多条线调用，但每条线必须写成不同版本。例如毕业照：
   - 宿舍修补线看同框资格和旧账。
   - 晚风线看亲密关系是否公开、谁看见、谁不问。
   - 社团女生派生感情线看该女生原本的社团任务、公开边界和生活半径如何挤压宿舍。
   - 兼职线看请假、迟到、错过不补。
   - 摆烂线看入口过期和低存在感。
   - 5X 外流线看旧人同场擦肩但不回旧线。

7. **回流只在线内回流**
   内回流点只能回到当前命运池的主事件，不允许绕出去后开启另一条命运线。回流可以改变关系温度、误读、好感、信任、资源代价和文本回声，但不能改变路线锁定结果。

8. **其它人物只读推进**
   没被选择为当前线核心的人，仍按母本推进。他们能提供现实压力和价值对照，但不能抢走当前线主轴。例如走唐骁线时，周屿可以作为活动圈压力出现，但不能临时开启周屿线。

9. **5X 仍是唯一硬外流**
   5X 不是普通软外流，而是唯一硬外流。进入后第六卷、第七卷只走 5X 单线；旧宿舍和旧人物只能作为擦肩、短句、只读群消息或固定毕业流程出现。

## 共享与外流范围

| 阶段 | 共享程度 | 允许内容 | 禁止内容 |
|---|---|---|---|
| 第一卷 | 全线共享 | 入校、4XX 初识、座位、宿管、家庭初信号、基础好感 | 不开长线外流 |
| 第二卷 | 全线共享 | 军训手机事件、规则感、公开边界、晚风初连线、宿舍初裂纹 | 不锁定完美 / 兼职 / 感情 / 室友线 |
| 第三幕 | 开始分流 | 社团、主持招新、活动圈、摄影、新闻中心、幕后物资、公开表达或回寝室，先触发 `POOL-A3-ACTIVITY-PUBLIC` 父池 | 不把融媒体或完美线误当成第三幕唯一根路线 |
| 第四幕 | 二次分流 | 暑假、生活费、兼职、快递站、打印店，触发 `POOL-R4-WORK` | 不把兼职线写成普通支线外传 |
| 第五幕前段 | 主轴亲密分流 | 晚风边界、游戏 / 微信 / 抖音 / 线下见面、亲密投入，触发 `POOL-R5-WANFENG` | 主轴不能直接跳沈嘉禾 / 夏知微等其它感情线 |
| 第五幕前段 | 社团女生派生感情 | 仅当 A3 父池中已有女生角色连续出现并形成生活半径时，才允许触发 `POOL-R5-ROMANCE` | 没有女生出现的社团线不得凭空生成其它女主 |
| 第五幕中后段 | 站队分流 | 周屿 / 唐骁 / 陆沉 / 不站队 / DEFAULT-4XX，触发对应宿舍命运池 | 不让玩家同时完整走三室友线 |
| 第五幕隐藏点 | 硬外流 | 5X 唯一硬外流，触发 `POOL-R5X-HARD` | 不回原多线 |
| 第六 / 七卷 | 池内承接 | 当前命运池版本的关系定形、毕业流程和结算 | 不重新开放其它命运线主分支 |

## 命运池字段建议

后续拆 JSON 时，建议在现有路线状态上增加命运池字段：

```json
{
  "canon_line": "MOTHER-V4",
  "shared_until": "act2_end",
  "branch_mode": "butterfly_outflow",
  "route_parent_pool_id": "POOL-R5-STAND",
  "route_pool_id": "POOL-R5-TANG",
  "route_focus": "tang",
  "route_focus_tag": "focus_tang",
  "route_mode_tag": "mode_rule",
  "child_outflow_id": "R5-TANG",
  "outflow_permission_class": "main_axis",
  "female_candidate_present": false,
  "romance_outflow_allowed": false,
  "allowed_next_routes": ["R5-STAND", "R5-WANFENG"],
  "outflow_stage": "act5_stand",
  "pool_entry_choice": "P0D_STAND_WITH_TANG",
  "active_route_id": "R5-TANG",
  "closed_route_ids": ["R5-ZHOU", "R5-LUCHEN", "R5-LIEFLAT", "R5-ROMANCE"],
  "grafted_lockpoints": ["ME-STAND-BURST", "ME-C407-DEMO", "ME-GRADUATION"],
  "soft_echo_routes": ["default_4xx", "activity", "romance"],
  "hard_outflow": false
}
```

| 字段 | 用途 |
|---|---|
| `canon_line` | 指向恒定母本线。 |
| `shared_until` | 标记共享剧情到第二卷结束。 |
| `branch_mode` | 标记本项目采用关键抉择蝴蝶效应外流，而不是全支线并行。 |
| `route_parent_pool_id` | 当前路线所属父池。第三幕应先进入 `POOL-A3-ACTIVITY-PUBLIC`，再由 focus / mode 分发子外流。 |
| `route_pool_id` | 当前命运池 ID，一条线一个池。 |
| `route_focus_tag` | 父池内部的主要通道，例如 `focus_hosting`、`focus_newsroom`、`focus_photo`、`focus_backstage`、`focus_volunteer`、`focus_dorm_return`。 |
| `route_mode_tag` | 父池内部的命运模式，例如 `mode_normal`、`mode_perfect`、`mode_pressure`、`mode_avoid`、`mode_dorm`。 |
| `child_outflow_id` | 父池确认后的子外流路线编号，例如 `R3-PERFECT`；未锁定时可为空。 |
| `outflow_permission_class` | 当前路线的外流权限层，例如 `main_axis`、`club_with_female_candidate`、`club_without_female_candidate`、`hard_5x`。 |
| `female_candidate_present` | 当前社团 / 活动半径中是否已经有明确女生角色以连续事件出现。只出现一次擦肩不算。 |
| `romance_outflow_allowed` | 是否允许进入晚风之外的其它感情线。只有 `club_with_female_candidate` 可为 true。 |
| `allowed_next_routes` | 当前池允许导向的下一组路线白名单，用于防止主轴无限外流。 |
| `a3_activity_choice_count` | 第三幕社团活动父池已完成的选择次数，范围为 0-3。 |
| `a3_focus_votes` | 第三幕三次选择里各 focus 的累计次数，用于判断最终靠近哪个社团 / 活动半径。 |
| `a3_mode_votes` | 第三幕三次选择里各 mode 的累计次数，用于判断正常参与、完美自控、压力、回避或回寝。 |
| `a3_join_result` | 第三次选择后锁定的社团归属或不进入结果，例如 `join_hosting`、`join_newsroom`、`join_photo`、`join_backstage`、`join_volunteer`、`join_none`。 |
| `pool_entry_choice` | 导致进入该池的关键抉择。 |
| `active_route_id` | 当前可写主线。 |
| `closed_route_ids` | 进入当前池后关闭的其它命运线。 |
| `grafted_lockpoints` | 本池需要嫁接的母本公共锁点。 |
| `soft_echo_routes` | 可作为软照面和文本回声出现的路线。 |
| `hard_outflow` | 是否进入 5X 硬外流。 |

## 第三幕父池与子外流

第三幕的根路线不是融媒体，也不是直接等同于完美线。原本逻辑是“社团招新、汇演、班会、主持、摄影、公开材料、幕后物资和回寝室”共同构成第一次生活半径分发。

因此第三幕先进入父池：

```text
POOL-A3-ACTIVITY-PUBLIC
```

父池只回答一个问题：林亦舟在第三幕以后，主要把时间、责任和公开风险交给哪里？

| focus | 主要通道 | 支撑人物 / 场景 | 说明 |
|---|---|---|---|
| `focus_hosting` | 主持队、九龙壁舞台、活动串场 | 周屿、秦越 | 靠近周屿和活动圈，容易产生人情债和公开场面压力。 |
| `focus_newsroom` | 新闻中心 / 融媒体、材料、采访、授权 | 许棠、明德楼材料窗口 | 处理公开叙事和授权边界，但它只是通道，不是父池本身。 |
| `focus_photo` | 摄影社、照片、镜头、公开边界 | 夏知微 | 处理“被看见”和“能不能公开”的边界。 |
| `focus_backstage` | 幕后物资、签到表、串场稿、归档 | 秦越、唐骁、沈嘉禾 | 处理流程、效率和自我管理。 |
| `focus_volunteer` | 青志协、志愿服务、秩序维护 | 活动 NPC、生活人物 | 处理服务型社交和责任感。 |
| `focus_dorm_return` | 回 4XX、处理作业、公共桌 | 周屿、唐骁、陆沉 | 活动没有成为主半径，宿舍继续是默认锚点。 |
| `focus_public_avoid` | 拒绝正脸、不上台、借热闹逃避 | 晚风、夏知微、无名生活人物 | 不进入公开半径，但会积累回避和错过。 |

父池内部再根据持续选择形成模式：

| mode | 形成条件 | 子外流结果 |
|---|---|---|
| `mode_normal` | 参与活动但不过度吞噬宿舍生活 | 只增加活动回声，不锁长线。 |
| `mode_perfect` | 连续选择流程、低风险、按时、公开可控、自我管理 | 形成 `R3-PERFECT` 子外流。 |
| `mode_pressure` | 连续接临时任务、救场、人情债、多头补位 | 形成活动压力子外流，可后续并入完美线或第五幕周屿线前置。 |
| `mode_avoid` | 连续拒绝曝光、不上台、短句逃开 | 形成回避前置，后续更容易进入摆烂或 5X 候选。 |
| `mode_dorm` | 连续回 4XX 当面处理关系 | 不锁第三幕外流，继续 DEFAULT-4XX。 |

### 三次社团活动机会

第三幕父池不做无限摇摆，也不靠一次点击锁死。它采用“第一次全开放、第二次维持或试新、第三次结算”的结构，更接近真实大学生报名社团时的犹豫过程。

| 次数 | 叙事功能 | 可选方向 | 判定作用 |
|---|---|---|---|
| 第一次 | 全开放试探 | 主持队、新闻中心 / 融媒体、摄影社、幕后物资、志愿服务、回 4XX、不参加公开活动都可选 | 记录 `a3_first_focus`，形成第一份兴趣或回避倾向，不锁线。 |
| 第二次 | 维持或试新 | 维持第一次方向，或去另一个方向试试；也可以回 4XX / 不参加 | 记录 `a3_second_focus`。如果与第一次相同，方向稳定；如果不同，形成两个候选方向。 |
| 第三次 | 结算 | 坚持第一个、坚持第二个、两个都要、两个都不要 | 结算 `a3_join_result` 与 `route_mode_tag`。 |

第三次结算规则：

- 坚持第一个：锁定 `a3_first_focus` 对应的加入方向。
- 坚持第二个：锁定 `a3_second_focus` 对应的加入方向。
- 两个都要：不开放双主线，自动归为 `mode_pressure`，形成活动压力子外流或后续压力前置；两个方向中只有一个作为主 focus，另一个进入软照面 / 人情债 / 任务冲突。
- 两个都不要：结算为 `join_none`，进入 `DEFAULT-4XX` 宿舍主轴；社团只保留短回声、错过感或后续照面。
- 如果第一次或第二次已经选择回 4XX / 不参加，第三次“坚持该选择”也视为 `join_none`。
- `R3-PERFECT` 只能在“坚持第一个”或“坚持第二个”后成立，且该方向不是 `join_none`，并且前两次 / 第三次处理方式持续偏流程、低风险、按时、公开可控。选择“两个都要”不能进入完美线。

多选时不按“选了哪个社团”锁线，而按三次选择后的责任承担判定。没有胜出的方向只保留为关系变量、台词回声、人情债或后续照面。

### 外流权限白名单

为防止第三幕以后所有路线继续互相打架，每个池子必须先判断“有没有资格往下一类完整线外流”。

| 当前池 / 来源 | 是否允许其它感情线 | 可导向路线 | 说明 |
|---|---|---|---|
| `DEFAULT-4XX` 主轴 | 否 | `R5-STAND`、`R5-WANFENG` | 主轴完整外流只保留宿舍站队和晚风线；沈嘉禾 / 夏知微等其它感情线不能从主轴凭空跳出。 |
| A3 有女生连续出现的社团线 | 是 | `R5-ROMANCE`、`R3-PERFECT`、`R5-STAND`、`R5-WANFENG`、`R5-LIEFLAT` | 只有女生角色在该社团 / 活动半径内连续出现、承担事件压力和选择后果，才允许成为其它感情线入口。 |
| A3 没有女生连续出现的社团线 | 否 | `R3-PERFECT`、`DEFAULT-4XX` / `R5-STAND`、`R5-WANFENG`、`R5-LIEFLAT` | 主持、幕后、志愿等若没有女生候选进入生活半径，就只能流向完美、自我管理、回宿舍修补 / 站队、晚风或摆烂。 |
| `R5X-HARD` | 否 | `R5X-HARD` 内部结算 | 5X 仍是唯一硬外流，不回原多线。 |

判定标准：女生角色必须在该线内至少拥有连续事件、现实任务、可选择互动和后续回声，才能算 `female_candidate_present = true`。单次照面、路过、群消息、母版回声不开放其它感情线。

## 外流线不是外传

后续所有 `R3-PERFECT`、`R4-WORK`、`R5-WANFENG`、`R5-ROMANCE`、`R5-ZHOU`、`R5-TANG`、`R5-LUCHEN`、`R5-LIEFLAT`、`R5X-HARD` 都按外流线处理。

它们不是“在母本旁边额外发生的一段故事”，而是林亦舟在关键抉择后进入的一种大学生活版本。公共锁点照常发生，但以本线视角重新排列压力、关系和代价。

示例：

- 玩家在第三幕选择活动圈，后续宿舍冲突仍发生，但林亦舟是从活动信用、公开边界和被工具化的角度进入。
- 玩家在第四幕选择兼职，后续站队仍发生，但林亦舟是从缺席、排班、钱压和被误读的角度进入。
- 玩家在第五幕选择唐骁，后续毕业照仍发生，但林亦舟是从规则、授权、项目归档和低温协作的角度进入。
- 玩家在第五幕不站队，后续毕业饭局仍发生，但林亦舟是从入口过期、没人再问和低存在感的角度进入。

## 开发示例

### 第五幕站队后进入唐骁线

玩家选择靠近唐骁后，系统不再同时开放周屿线、陆沉线、摆烂线和专注感情线的完整主分支。

后续写法应为：

- 当前主池：`POOL-R5-TANG`。
- 主事件：C407、评分细则、项目演示、归档、公共桌边界。

## 防爆炸规则

| 风险 | 错误写法 | 正确写法 |
|---|---|---|
| 支线交叉爆炸 | 走唐骁线时又能完整追周屿线、晚风线、兼职线 | 其它人只做软照面，不开完整主分支 |
| 公共事件重复爆炸 | 每条线都复制同一段毕业照剧情 | 毕业照作为锁点，各线写不同版本目标 |
| 变量池污染 | 所有路线共享所有好感和结局变量 | 每条线有本地变量，只保留少量全局变量 |
| 回流变新路线 | 出去聊一次夏知微后转进夏知微线 | 夏知微只提供影像/授权/视角回声，回到当前池 |
| 母本被玩家改写 | 未选择的人物停止成长或改性格等玩家 | 未选择人物按母本生活，林亦舟只是错过或低频接触 |

## 后续开发顺序

1. 先保留小说母本作为 `MOTHER-V4`。
2. 将第一卷、第二卷作为所有路线共享序章。
3. 按第三幕、第四幕、第五幕关键抉择点建立 `route_pool_id`。
4. 每条命运池只写自己的主事件链和内回流点。
5. 每个公共锁点按当前池生成一个版本，而不是共用同一段可选树。
6. 第七卷只读取当前池长期累积的关系、代价和错过，生成毕业温度。

## 结论

本项目后续不采用“所有支线挂在一条主线上”的树状堆叠，而采用“第一二幕共享 + 关键抉择外流 + 命运池并行”的结构。玩家每次重大选择不是把所有人的人生重新分叉，而是把林亦舟推入一种新的生活池。池内可以丰富、可玩、有回流、有内部支线，但池与池之间不互相抢主线。
