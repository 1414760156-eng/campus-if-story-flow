# 开发细稿_IF可玩支线DEFAULT-4XX内回流剧情页

## 用途

本文是 `DEFAULT-4XX` 宿舍修补线的内回流剧情页样板，不是小说正文，也不是试玩器脚本。它用于指导后续拆分 JSON、剧情页、选项页和变量回调。

本文件先完成早中段五个基础回流点：

- `RET-DEFAULT-01` 公共桌外卖：处理“替不替人点 / 留不留口味 / 饭凉了谁负责”的饭桌温度。
- `RET-DEFAULT-02` 楼下不进门：处理“回避现场 / 当面回去 / 被现实流程推回”的核心玩法。
- `RET-DEFAULT-03` 凌空栈道软照面：处理“外部照面 / 边界提醒 / 消息拉回”的软回流。
- `RET-DEFAULT-04` 快递站旧包裹：处理“代取 / 退回 / 错件返工”的物品边界。
- `RET-DEFAULT-05` 事项群短句：处理“群聊短句 / 私聊误读 / 撤回消息 / 授权边界”的核心玩法。

所有回流方向都必须满足：

1. 至少 3-4 个连续剧情拍点。
2. 每个拍点都有玩家可操作的动作或对话选择。
3. 回流只影响人物关系、好感 / 信任、误读、微心态和后续文本语气。
4. 不改变母版主线锁点，不开启新路线，不替任何人改命。

## 共用页面字段建议

正式拆 JSON 时，每个剧情页建议使用以下字段：

```json
{
  "route_id": "DEFAULT-4XX",
  "return_id": "RET-DEFAULT-02",

  "beat_id": "RET-DEFAULT-02-A03",

  "forbidden_effect": "不得直接和解，不得开启唐骁个人线"
}
```

## `RET-DEFAULT-01` 公共桌外卖

### 回流定位

触发条件：第四章饭桌回声或任意公共桌低温段，玩家选择用点饭、取饭、留饭来缓和宿舍气氛。

核心功能：把“吃点东西吧”做成可操作的关系测试。饭桌可以增温，也可能暴露替人决定、默认口味、迟到和没人回应的问题。外卖不能解决旧账，只能让玩家看见谁还愿意坐下、谁只是拿了东西就走。

禁止写法：

- 不允许一顿饭直接修复宿舍关系。
- 不允许把“我给你点了”写成天然体贴，必须检查本人确认。
- 不允许用口味偏好替代人物冲突。

## `RET-DEFAULT-02` 楼下不进门

### 回流定位

触发条件：玩家在第一章选择 `DEFAULT-CH01-C02`，只回“收到”，先不进门。

核心功能：让“暂时不进门”变成可玩的回避段，而不是一句旁白。玩家可以选择同质回流、流程催回或异质软照面，但无论怎么绕，最后都必须回到 4XX 公共桌或门口，继续宿舍修补主事件。

禁止写法：

- 不允许写成“林亦舟想了想还是进去了”。
- 不允许夏知微、许棠替宿舍矛盾给答案。
- 不允许唐骁在楼下直接道歉或替其他室友解释。

| 第3拍：玩家选择 | `RET-DEFAULT-05-B03` | 周屿又发：“别搞得太认真，怪尴尬的。”这一次他没有撤回。 | A. 顺着玩笑说一句轻的；B. 说“尴尬也比误会继续好”；C. 停在这里，不继续逼。 | A：`dorm_warmth +1`, `old_debt` 不降；B：`repair_depth +1`, `conflict_heat +1`；C：`zhou_trust +1`, `late_regret +1`。 |
| 第4拍：回流与结算 | `RET-DEFAULT-05-B04` | 林亦舟回到公共桌时，周屿把手机扣在桌上：“行，我自己填。”他说得像开玩笑，但笔已经拿起来。 | A. 不提私聊，只让他填；B. 当面确认“你愿意自己填？”；C. 把他的玩笑拿出来缓和气氛。 | 回到 `DEFAULT-CH02-N03`。A：`public_boundary +1`；B：`zhou_trust +1`；C：`misread +1`，唐骁可能觉得被绕开。 |

对话基调：

- 周屿不直接说“我受伤了”，他会先说“我适合当例子”。
- 玩家越急着证明自己不是审判，周屿越会把话绕开。
- 私聊能增温，但也会让其他人觉得公共桌被拆成小圈。

### 方向 C：表情沉默回

方向功能：陆沉用撤回和停顿表达边界。玩家能不能不替他保存、不替他解释，是这条方向的核心。

| 拍点 | 页面 ID | 场景与剧情 | 可操作选择 | 变量与回声 |
|---|---|---|---|---|
| 第1拍：现实触发 | `RET-DEFAULT-05-C01` | 群里出现“陆沉 撤回了一条消息”。撤回前，林亦舟只看见半句：“我自己填……”公共桌上没人立刻提。 | A. 当面问他；B. 截图保存撤回提示；C. 假装没看见。 | A：进入边界确认；B：`proxy_risk +1`；C：`late_regret +1`。 |
| 第2拍：照面或打断 | `RET-DEFAULT-05-C02` | 陆沉把手机放到桌边，声音不高：“不用截。我没说完。”唐骁看向林亦舟，周屿也没插话。 | A. 说“那你自己补”；B. 问“哪些不能写”；C. 解释“我不是想留证据”。 | A：`luchen_trust +1`；B：`public_boundary +1`；C：`luchen_trust -1`，解释过多。 |
| 第3拍：玩家选择 | `RET-DEFAULT-05-C03` | 表格里陆沉那一行还空着。他把手放在键盘上，停了两秒：“别写我没意见。” | A. 删除“默认同意”字样；B. 让他自己打完整句；C. 先留空，等他晚点填。 | A：`public_boundary +1`；B：`luchen_trust +1`, `time_debt +1`；C：`dorm_trust +1`, `project_stability -1`。 |
| 第4拍：回流与结算 | `RET-DEFAULT-05-C04` | 陆沉最后只打了六个字：“本人确认到此。”句子很短，但没有再撤回。公共桌上没人笑。 | A. 保存表格；B. 问是否还要补一句；C. 不再追问。 | 回到 `DEFAULT-CH02-N04`。A：`project_stability +1`；B：`luchen_trust -1`，越界风险；C：`luchen_trust +1`, `repair_depth +1`。 |

对话基调：

- 陆沉的边界不靠长台词，靠“别写我没意见”这种具体短句。
- 林亦舟不能把撤回消息当证据，也不能把沉默当授权。
- 这条方向适合降低 `proxy_risk`，但会增加时间成本。
