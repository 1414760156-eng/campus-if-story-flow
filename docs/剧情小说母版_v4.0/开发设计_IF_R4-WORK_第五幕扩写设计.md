# IF R4-WORK 第五幕扩写设计

更新时间：2026-05-29

## 目标

第五幕不能只是第四幕后面的任务延长线。它应该是 `R4-WORK` 最核心、最可玩的卷：第四幕把暑假去向、本人签字、钱、补考、联系人和留校事实压到纸面上；第五幕要让这些纸面事实开始改变相处、缺席、晚班、活动、项目和第六幕入口。

设计目标：

- 体量明显超过第四幕。
- 选择不是“先做哪张表”，而是“他这一次站在哪边”。
- 校园人物不再只是背景名字，必须真正造成时间冲突、公开压力或关系回声。
- 工作线仍留在 `POOL-R4-WORK`，不打开晚风线、活动线、学业线或 5X 外流线。
- 第五幕结尾必须沉到第六幕：稳定、修补、低温、固定班锁死或低完成风险。

## 设计选择

### 方案 A：保留 12 窗，只在每窗内扩写

优点是改动小，当前试玩结构稳定。缺点是玩家仍可能觉得第五幕和第四幕体量接近，很多回流只能塞进反馈页。

不推荐作为最终方案。

### 方案 B：扩成 15 个主选择窗，并补回流层

在现有 12 窗基础上新增 3 个核心冲突窗：班前门口回流、饭点试运行、期末复发夜。原有 12 窗不推倒，只重排和扩写，让第五幕从“连续处理事情”变成“返校第一周不断站队”。

推荐采用。

### 方案 C：重建为 20+ 节点母本级大图

最接近 5A 母本厚度，但当前风险太高，会拖慢 4->5->6 接口验证，也容易再次变成复杂但不可读的表格工程。

暂不推荐。

## 推荐方案

采用方案 B：`15 个主选择窗 + 4 个软回流组 + 2 个母本级波峰`。

第五幕不是从 12 窗机械加到 15 窗，而是把现有结构改成三段：

1. **入口段：纸面事实变成当面事实**
   B01-B04。晚班、4XX、C407、晚风和活动群第一次在同一晚撞上。

2. **扩散段：外部人物开始真正消耗他**
   B05-B10。项目、家庭证明、秦越活动名单、夏知微照片、许棠公开材料、沈嘉禾格式边界都不解决问题，只把问题从“自己忙”推成“别人要怎么处理他”。

3. **复发段：固定班成为新生活半径**
   B11-B15。饭点、学习债、书屋安静处、固定班低温、第六幕入口依次兑现。

## 四个软回流组

这些不是新路线，只是短离场后回到工作线。

| 回流组 | 落点 | 玩法作用 |
|---|---|---|
| 班前门口回流 | B01-B04 | 玩家决定今晚第一句是给 4XX、快递站，还是留在楼梯口；对应“门口空间选择”。 |
| 沈嘉禾 / 打印店格式回流 | B02-B11 | 她只能帮看格式，不能替他写理由；打印店老板用规则把材料推回给本人。 |
| 秦越 / 许棠活动公开回流 | B08-B12 | 活动名单和公开材料同时消耗他，可靠不是情绪，是名字会不会被写上。 |
| 晚风 / 夏知微夜晚回流 | B05-B09/B13 | 晚风要求提前知道，夏知微照片照见缺席；两者都不替主线解决。 |

## 两个母本级波峰

### 波峰一：工作线规则会

落点：B06-B08。

C407、快递站、4XX、活动名单和证明材料第一次形成公开规则冲突。不是每个人都讲道理，而是每个人都拿着一张可以查到他名字的东西。

玩家要决定：

- 把排班优先写清。
- 把 C407 可交付写清。
- 把 4XX 当面缺席说清。

后果不是立刻吵架，而是下一次谁还会不会默认等他。

### 波峰二：期末 / 补考复发夜

落点：B10-B11。

工作后饭点、学习债、晚班和补考材料在同一晚复发。第四幕里“先补资料 / 先接班 / 先说清”的旧选择要在这里被检验。

玩家要决定：

- 少接一晚保学习。
- 接完班再补学习债。
- 把无法完成的部分提前说清。

后果直接进入 B14-B15 的固定班半径和第六幕入口。

## 人物使用规则

| 人物 | 第五幕作用 | 禁止事项 |
|---|---|---|
| 周屿 | 用玩笑、饭盒、活动照面测出他还算不算 4XX 的人 | 不替林亦舟解决晚班或证明 |
| 唐骁 | 用表格、C407、可交付项逼他写清楚 | 不变成单纯催任务 NPC |
| 陆沉 | 用门禁、勤工规则、常温水提示边界 | 不替他解释情绪 |
| 晚风 | 要求提前知道，不再接受“到时候看” | 不打开完整晚风线 |
| 夏知微 | 照片和授权照出缺席被看见 | 不变成恋爱支线替代 |
| 沈嘉禾 | 格式、文档、图书馆座位，只看能否提交 | 不安慰，不代写 |
| 秦越 | 活动名单和物资交接真正占用时间 | 不只是群消息背景 |
| 许棠 | 公开材料和命名口径决定谁被写进去 | 不替主角洗白 |
| 老师 / 助教 | 规则压力，只问材料和截止 | 不解释主题 |
| 打印店老板 / 宿管 / 食堂窗口 / 快递站负责人 | 通过重复照面让校园真实存在 | 不变成旁白说明工具 |

## 选择页写法

选择页必须像现场逼出的下一步，不写功能解释。

可以写：

- “七点二十快到了，门还半开着。”
- “饭盒还盖着，窗口阿姨已经认出他。”
- “名单上有他的名字，旁边那一栏还空着。”

不写：

- “本窗决定工作线倾向。”
- “这是活动回流点。”
- “该选择会影响第六幕稳定度。”
- “三类压力并列出现。”

## 验收标准

第五幕扩写完成后，试玩应满足：

- 主选择窗达到 15 个。
- 每窗至少有一个现场动作，不以抽象压力开场。
- 至少 6 个校园人物在选择或反馈中真正造成后果。
- 至少 4 个软回流组能被玩家感到“短离场又回来了”。
- 至少 2 个波峰让玩家感到第五幕明显比第四幕更重。
- 不出现开发词、变量词、路线说明。
- `ACT6-WORK-B01` 能读取稳定 / 修补 / 低温 / 固定班锁死四类后果。

## 执行顺序

1. 先补 B04、B10、B15 三个新增主选择窗的结构设计。
2. 再重排现有 B01-B12 的编号和 `next_block`。
3. 写 B01-B05 第一批正文，导出试玩确认口味。
4. 写 B06-B10 第二批正文，确认波峰一和饭点试运行。
5. 写 B11-B15 第三批正文，确认期末复发和 Act6 入口。
6. 全量导出 4->5 试玩，做一次玩家侧审读。
