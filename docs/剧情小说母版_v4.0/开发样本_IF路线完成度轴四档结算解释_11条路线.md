## 审读重点

- 代理家庭轴不能凭压力代理变量给出完美家庭轴。
- 软回声轴可拉低温度，但不能单独抢走路线主题。

- 5X 完美是清醒离开，不是隐藏真结局。

## A3-ACTIVITY-PUBLIC｜活动父池

- 玩家侧标题：把名字写到公开事务里
- 核心判定：公开信用能不能被林亦舟自己承担，而不是变成逃离宿舍和私人解释的漂亮外壳。
- 不应写成：单纯社团成功线、恋爱候选入口合集、或跳过宿舍旧账的捷径。

### 完美结局：公开信用成立，但不吞掉宿舍和私人边界

- 输入画像：玩家连续处理授权、署名和活动归档，也在第五幕向 4XX 说明缺席原因，候选来源只作为合法来源保留。
- 轴结果：interest_activity=完美结局；dorm_relationship=完美结局；study_or_work=完美结局；romance_relationship=良好结局；self_boundary=完美结局
- 估算路线分：87.5
- 玩家可见解释：这条结算把 A3 判为完美，是因为公开材料真正落到了林亦舟自己名下，活动信用没有变成逃离公共桌的漂亮理由。宿舍、学业和候选来源都留下了边界说明，所以热闹没有继续外溢成误读。
- 公式验证点：验证活动强轴完美时，宿舍缺席轴不能因为曾缺席而自动拉低整条路线。

### 良好结局：活动做成了，压力也扩散了一部分

- 输入画像：玩家完成活动任务和公开范围说明，但宿舍旧账只处理了一部分，项目和私人解释仍被活动时间挤压。

- 玩家可见解释：这条结算是良好而不是完美：公开信用成立，玩家也没有把活动写成恋爱入口合集，但第五幕之后仍有压力扩散。4XX 知道事实，却没有真正得到完整解释。
- 公式验证点：验证公开信用正向变量可以支撑良好，同时宿舍和学业次轴成本仍可见。

### 一般结局：材料能交，但边界主要靠低风险处理

- 输入画像：玩家保住活动交付和最低归档，公开范围只做保守处理，宿舍只收到结果，候选来源没有充分说明。

- 估算路线分：50
- 玩家可见解释：一般结局表示功能性通过：活动材料交上去了，公开事故也没有爆掉，但玩家没有真正承担公开信用背后的解释成本。宿舍停在礼貌距离，候选来源只被记录，没有形成清楚回声。
- 公式验证点：验证强轴正常通过时，路线不应被误读成失败，只是温度偏低。

### 遗憾结局：热闹盖住了边界

- 输入画像：玩家用活动正确性绕开宿舍解释，公开材料和候选来源被误读，学业材料也被活动证明挤在一起。

- 核心判定：生活费、排班、工资和家庭压力是否被真实处理，同时不让关系完全被现实时间切碎。

- 公式验证点：验证 strong 轴 normal 时路线保持一般，不被工作完成度单独抬高。

### 遗憾结局：收入保住了，人被时间切开

- 公式验证点：验证缺席封顶和压力未解释能共同把路线压到遗憾。
