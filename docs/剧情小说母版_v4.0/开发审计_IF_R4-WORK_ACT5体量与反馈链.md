# IF R4-WORK Act5 体量与反馈链审计

更新时间：2026-05-26

## 审计结论

当前 `R4-WORK` Act5 已经完成结构接通，但还不是完整可玩第五幕。

现有 `act5_bridge_nodes` 的 5 个选择窗 / 15 个方向，只能算“第五幕五类主压力窗口”的宏观桥接骨架：它能把 `P0-B -> Act4 -> Act5 -> Act6` 接起来，也能把变量和 `act6_echo_hook` 带到后续，但玩家侧体验仍会偏成“看 4 页摘要 -> 点一个方向 -> 进入下一节点”。这不符合已冻结的 Act4 样板，也不符合支线进入独立池后仍需保持母版主轴玩法密度的规则。

因此，Act5 的正确交付口径不是 5 个选择，而是：

- 5 个宏观压力窗口。
- 15 条方向反馈链。
- 每条方向至少有选择后剧情反馈。
- 关键方向补微心态选择。
- 玩家侧选项必须写成林亦舟真实会发出、说出、写下或做出的具体动作，不能写成“沟通策略”或变量解释。

## 未满足项

- 没有方向选择后的玩家侧剧情反馈页。
- 没有微心态组和微选项。
- 当前方向 `label` 仍偏摘要，部分表达像策划标签，而不是玩家会点下去的具体动作。
- 选择后没有足够的“对话、动作、物件、场景后果”，会造成点击后直接跳下一题的问答感。
- Act5 还不能作为其它支线第五幕复刻样板。

## 正确体量口径

| 项目 | 当前 | 最低完成口径 | 推荐样板口径 |
|---|---:|---:|---:|
| 宏观压力窗口 | 5 | 5 | 5 |
| 宏观方向 | 15 | 15 | 15 |
| 方向反馈链 | 0 | 15 | 15 |
| 选择后反馈页 | 0 | 15 | 30 |
| 微心态组 | 0 | 15 | 20 |
| 微选项 | 0 | 45 | 60 |
| 玩家可感知选择层 | 5 | 约 20 | 约 25 |

解释：

- “方向反馈链”指玩家选 A / B / C 后进入的连续剧情反馈，不是变量字段。
- 最低口径是每个宏观方向至少 1 段选择后反馈，并配 1 组微心态；推荐口径是关键方向扩成 2 段反馈和 2 组微心态。
- 每组微心态按 3 个玩家侧具体动作计算，因此 15 组为 45 个微选项，20 组为 60 个微选项。
- Act5 作为支线进入独立池后的第一整幕，不应比 Act4 断崖式变薄。

## 分节点缺口

### `ACT5-WORK-B01` 宿舍站队从排班缺席开始

当前三方向：

- 固定班照接，给 4XX 最低事实。
- 请假半小时，当面解释钱压。
- 不讲家里，只说今晚有班。

缺口：

- 需要把公共桌、晚班排班、唐骁追问、4XX 误读、勤工值班表写成选择后反馈链。
- 至少要有“怎么把今晚说出口”的微心态选择。
- 目标不是打开宿舍站队线，而是让宿舍关系作为 `POOL-R4-WORK` 内的压力回声进入 Act6。

### `ACT5-WORK-B02` 亲密 / 晚风低频解释

当前三方向：

- 把排班和回寝时间说给晚风。
- 只说有班，不解释更多。
- 先回 4XX 说明不是躲人。

缺口：

- 需要把晚风消息、光谷时间、宿舍回寝、手机回复写成具体动作。
- 微心态选项应是“发哪一句、改哪一版、留哪个时间”，不能写成“保信任 / 保距离”。
- 晚风只能作为工作池内的亲密压力，不打开晚风线。

### `ACT5-WORK-B03` 项目 / 规则与证明材料

后续状态：`ACT5-WORK-B03` 的 3 条方向反馈链、7 个反馈页、4 组微心态和 12 个微选项已补入源文档；以下缺口保留为正式 JSON 写回前的核对项。

当前三方向：

- 先补勤工证明和收入说明。
- 先补 C407 项目分工。
- 写清可交付和不可到场。

缺口：

- 需要补打印店、明德楼窗口、C407 材料、唐骁 / 陆沉边界的选择后反馈。
- 微心态选项应围绕“证明解释到什么程度、项目分工写到哪里、哪些不能代签”展开。
- 规则不是系统说明，要落到林亦舟手里的材料、签字和消息。

### `ACT5-WORK-B04` 活动 / 兴趣与工作生活半径

后续状态：`ACT5-WORK-B04` 的 3 条方向反馈链、7 个反馈页、4 组微心态和 12 个微选项已补入源文档；以下缺口保留为正式 JSON 写回前的核对项。

当前三方向：

- 固定班成为主生活半径。
- 降低班次，保一次宿舍照面。
- 给排班、活动、作业都写停止条件。

缺口：

- 需要补活动群、值班表、宿舍门口照面、作业截止和排班交换的反馈链。
- 微心态选项应围绕“这周到底怎么排”“哪一次照面必须保留”“停止条件写给谁”展开。
- 这一节点要让玩家看到工作线生活半径形成，而不是只看到“倾向增强”。

### `ACT5-WORK-B05` 回避 / 外流门槛关闭

后续状态：`ACT5-WORK-B05` 的 3 条方向反馈链、7 个反馈页、4 组微心态和 12 个微选项已补入源文档；以下缺口保留为正式 JSON 写回前的核对项。

当前三方向：

- 留在工作线，写清第一周安排。
- 保留宿舍回声，但不回默认线。
- 拒绝 5X / 摆烂诱因，按现实清单走。

缺口：

- 需要补许澈、新圈子、宿舍沉默、摆烂诱因、第一周清单的选择后反馈。
- 微心态选项应围绕“第一周清单发给谁、哪些诱因不接、如何保留宿舍回声但不回线”展开。
- 该节点是 Act5 -> Act6 的门槛，不应只用标签关闭外流。

## 建议 JSON 增补结构

正式写回 JSON 时，建议在每个 `act5_bridge_nodes[]` 下补：

```json
{
  "feedback_chains": [
    {
      "direction_id": "ACT5-WORK-C01-A",
      "feedback_pages": [
        {
          "page_id": "ACT5-WORK-C01-A-F01",
          "location": "青枫居 4XX / 公共桌",
          "text_brief": "玩家侧剧情反馈，不写变量解释。"
        }
      ],
      "micro_groups": [
        {
          "micro_id": "ACT5-WORK-C01-A-M01",
          "prompt": "玩家侧心态抉择标题。",
          "choices": [
            {
              "choice_id": "ACT5-WORK-C01-A-M01-1",
              "label": "林亦舟真实会发出的具体话或动作。",
              "text_brief": "选择后的短反馈。",
              "variable_delta": {}
            }
          ]
        }
      ]
    }
  ]
}
```

字段原则：

- 玩家正文只放现场剧情和可点击动作。
- 变量、路线池、`act6_echo_hook`、完成条件只放结构字段，不进入玩家文本。
- 微心态点击后直接进入反馈，不做二次确认。
