# 开发长线_IF可玩支线DEFAULT-4XX宿舍修补线

## 用途

本文用于把 `DEFAULT-4XX` 从 P0-D 的站队分歧骨架扩展成可直接拆成剧情页、选项页、变量回调和结算条件的长线开发模板。

它不是小说正文，也不是支线外传。它只规定：玩家选择让林亦舟继续把生活重心留在青枫居 4XX 时，哪些事件会变得更密、更可玩，哪些回流点可以短暂绕出去再回到宿舍修补主线，哪些变量会影响第六卷协作温度和第七卷毕业结算。

## 路线定位

路线编号：`DEFAULT-4XX`

路线名称：宿舍修补线

路线等级：S 级主轴等量线

核心体验：林亦舟不逃去外部关系，不单押某一个室友，也不把矛盾拖到毕业。他选择继续留在 4XX 的旧账里，把事实、时间、缺席、误读、难听话和未说完的部分一次次放回同一张公共桌。

重点不是圆满。修补线只提高“还能当面处理”的概率，不保证所有人和解，不保证饭局坐满，不保证毕业照公开，也不保证离校后还频繁联系。

## 进入条件

建议满足以下条件进入本线：

| 条件 | 建议值 | 说明 |
|---|---:|---|
| `stand_conflict` | `>= 2` | 第五幕宿舍强冲突已经集中爆发 |
| `old_debt_visible` | `>= 2` | 旧账不再是背景，而是能被具体说出来 |
| `dorm_repair` | `>= 2` | 玩家至少两次选择当面处理或回公共桌 |
| `avoidance` | `<= 2` | 玩家没有持续静音、绕开、晚回、不解释 |
| `route_lock` | `false` | 未进入 5X 唯一硬外流 |

入口建议节点：`D_00_SHORT_GROUP` 事项群变短。

入口关键选择：

- `DEFAULT-ENTRY-01`：回群说“等我回去，当面说”。
- `DEFAULT-ENTRY-02`：不私聊任何一人，先回公共桌。
- `DEFAULT-ENTRY-03`：承认自己也想逃，但仍回去。

## 不改变的母版锁点

以下事件仍按母版发生，只改变林亦舟在其中的位置、亲疏和代价：

- 第五幕宿舍强冲突必然发生。
- 周屿、唐骁、陆沉都不能被写成单纯过错方。
- 第六卷 C407 项目、课程、材料归档和关系定形仍发生。
- 第六卷四人不再默认天天同行，宿舍退位仍发生。
- 第七卷预登记、毕业活动材料、开题 / 答辩、毕业照、清寝、饭局、离校手续仍发生。
- 5X 仍是唯一硬外流，普通绕出只允许软回流。

## 内回流硬规则

本线所有 `RET-DEFAULT-*` 都不是单线支路，也不是“出去一个选项再回来”。每个回流点必须同时满足以下标准：

1. 至少两个回流方向：同质回流、异质软照面、生活任务回流、消息回流、流程催回等可以组合，但不能只有一条固定返回路径。
2. 每个回流方向必须拆成三到四个连续剧情拍点。推荐四拍为：现实触发 -> 照面 / 打断 -> 对话选择 -> 带回声回流。
3. 每个拍点至少提供一个可操作动作或对话分歧，例如回不回、问不问、等不等、聊不聊刚发生的事、是否替别人解释、是否把话停在边界内。
4. 回流只影响人物关系、好感 / 信任、误读、微心态和后续文本语气。时间、钱、精力、迟到、排队、取件等现实内容可以作为当场代价或压力，但不能让普通回流变成新的路线锁定。

开发时不要把“回流方式一 / 二 / 三”直接当成可交付剧情页。它们必须继续展开成可玩的连续页面。最低交付结构如下：

| 层级 | 必须写清 | 示例 |
|---|---|---|
| 方向 | 这一趟绕出去会从哪种关系角度回来 | 和室友一起回 / 被宿管催回 / 碰到夏知微后回 |
| 第 1 拍 | 现实触发 | 不想进门、外卖超时、事项群催表、快递站取件 |
| 第 2 拍 | 照面或打断 | 碰到唐骁买夜宵、周屿从彩排回来、陆沉排班、夏知微拍夜景 |
| 第 3 拍 | 玩家对话选择 | 聊生活小事 / 聊刚才的冲突 / 沉默同行 / 问一句边界 |
| 第 4 拍 | 回到主线并留下回声 | 一起上楼、门禁催回、群消息截断、回公共桌时语气变化 |

示例：`RET-DEFAULT-02` 楼下不进门不能只写“遇到室友后回寝”。至少应拆为：

- 方向 A：同质回流。林亦舟坐在青枫居楼下不进门，碰到出来买夜宵的唐骁；玩家可选“只问买什么”“问刚才表格是不是太硬”“沉默跟他去小卖部”；最后两人一起回 4XX，结算 `tang_trust`、`dorm_warmth`、`avoidance`，不改变主线冲突是否发生。
- 方向 B：异质软照面。林亦舟绕到凌空栈道，碰到夏知微拍夜景；玩家可选“聊近况”“聊未来去向”“聊刚才没进门的原因”；最后被 4XX 事项群或门禁时间拉回，结算 `mind_clarity`、`public_boundary`、`avoidance`，夏知微仍按母版生活线推进。
- 方向 C：流程催回。宿管阿姨在门口登记夜归，玩家可选“解释马上回”“帮别人带一句”“不解释直接上楼”；最后回到 4XX 时，关系温度不一定上升，但下一次门口 / 群消息语气发生变化。

## 核心变量

| 变量 | 初始建议 | 上升含义 | 下降或过高风险 |
|---|---:|---|---|
| `dorm_trust` | 0 | 四人愿意继续当面核对事实 | 低值时只剩事项群短句 |
| `dorm_warmth` | 0 | 仍愿意坐下、等一下、留位置 | 高值也不等于和解，只表示温度可见 |
| `avoidance` | 0 | 林亦舟持续回避 | 高值会滑向摆烂线或低温毕业 |
| `old_debt` | 0 | 旧账被看见、被提及 | 高值会增加复发强度 |
| `repair_depth` | 0 | 不是压住矛盾，而是能说到根上 | 低值只形成表面平稳 |
| `proxy_risk` | 0 | 替别人说话、替别人处理物品的风险 | 高值影响陆沉和公开边界 |
| `public_boundary` | 0 | 照片、材料、群消息是否授权清楚 | 低值影响毕业照和主题材料 |
| `project_stability` | 0 | 第六卷专业协作稳定度 | 高值不自动增加亲密 |
| `graduation_temperature` | `unresolved_but_present` | 毕业时关系温度 | 由前述变量共同决定 |

## 章节体量与节奏

建议 10 章，约 80,000 到 120,000 字开发体量；作为文本开发稿时，每章至少拆 4 个有效选择、2 个变量回调、1 个内回流点或母版事件回声。

| 章 | 标题 | 母版事件 | 核心任务 | 关系压力 |
|---:|---|---|---|---|
| 1 | 事项群变短 | `ME-4XX-GROUP` | 从群短句回到当面 | 谁还愿意等他说第一句话 |
| 2 | 公共桌事实表 | `ME-STAND-BURST` | 列事实、删定罪词、确认时间 | 事实会不会被当审判 |
| 3 | 私下补话 | `ME-NOT-ENTER-DOOR` | 先补谁、怎么补、补到哪里停 | 补一个人就会让另一个人等 |
| 4 | 饭桌回声 | `ME-DAILY-CAMPUS` | 外卖、食堂、商业街形成回温或误读 | 坐下不等于说开 |
| 5 | 项目与材料协作 | `ME-C407-DEMO` | 专业协作恢复接口 | 专业稳定不等于亲密恢复 |
| 6 | 复发 | `ME-STAND-BURST` | 新事把旧账重新带出 | 旧解释会恶化新行为 |
| 7 | 毕业照预览 | `ME-GRADUATION` | 同框、授权、留存边界 | 同框不等于公开，不公开也不是否定 |
| 8 | 清寝 | `ME-GRADUATION` / `ME-EXPRESS-WORK` | 物品归属、代收代丢、宿管检查 | 谁有权处理别人留下的东西 |
| 9 | 最后一顿饭 | `ME-GRADUATION` / `ME-DAILY-CAMPUS` | 过时不补的人数确认 | 留座、迟到、错过和不替人填 |
| 10 | 武生院站 | `ME-GRADUATION` | 离校手续和最后短句 | 说开、淡化、体面远离或不和解 |

## 章节开发详表

### 第一章：事项群变短

章节目标：建立本线语气。4XX 从聊天群变成事项群，玩家的第一批选择决定林亦舟是把问题带回公共桌，还是继续把矛盾交给群消息消耗。

现实触发：

- 4XX 群里只剩“表格谁交”“外卖谁拿”“今晚回不回”。
- 公共桌上压着课程分工表、外卖小票、借物清单和未确认材料。
- 周屿在线但不接话，唐骁只发截止时间，陆沉只回“收到”。

可拆节点：

| 节点 ID | 节点功能 | 场景 | 主要人物 |
|---|---|---|---|
| `DEFAULT-CH01-N01` | 读取事项群短句 | 微信群 / 4XX 门口 | 四人 |
| `DEFAULT-CH01-N02` | 决定是否当面回去 | 青枫居走廊 / 楼下 | 林亦舟 |
| `DEFAULT-CH01-N03` | 第一次公共桌落座 | 4XX 公共桌 | 四人 |
| `DEFAULT-CH01-N04` | 选择第一句话 | 公共桌 | 四人 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `DEFAULT-CH01-C01` | 回群：“等我回去，当面说。” | `dorm_repair +1`, `avoidance -1` | 唐骁停止继续发长表，周屿没接玩笑 | 第二章默认公共桌开启 |
| `DEFAULT-CH01-C02` | 只回“收到”，先不进门 | `avoidance +1`, `old_debt +1` | 陆沉也只回一个字，群安静 | 触发 `RET-DEFAULT-02` 楼下不进门 |
| `DEFAULT-CH01-C03` | 私聊其中一人，问他现在能不能说 | `selected_private +1`, `dorm_warmth -1` | 被私聊者温度上升，另两人误读 | 第三章补话对象偏移 |
| `DEFAULT-CH01-C04` | 把手机扣下，直接推门 | `dorm_trust +1`, `conflict_heat +1` | 屋里没人立刻开口 | 第二章冲突起点更高 |

内回流点：

- `RET-DEFAULT-02` 楼下不进门。
- 同质回流方式一：遇到出来买夜宵的室友，只聊“今晚吃什么”，不聊冲突，回去后 `dorm_warmth +1` 但 `old_debt` 不降。
- 同质回流方式二：被宿管阿姨提醒别在楼下拖到门禁，带着现实压力回屋，`avoidance -1`，但语气更硬。
- 异质软照面：绕到凌空栈道短暂碰见夏知微拍夜景，只获得一句“别把照片当证据”，不改变夏知微主线，`public_boundary +1`。

对话逻辑：

- 周屿先用“这么正式啊”挡一下，不主动认错。
- 唐骁只问“先说哪件”，不接情绪话。
- 陆沉不开头，但会观察林亦舟有没有替他说。

防漂移：

- 不允许第一章直接和解。
- 不允许让任何一名室友成为旁白式心理说明者。
- 群消息必须短，靠短句、撤回、已读不回制造压力。

### 第二章：公共桌事实表

章节目标：把宿舍矛盾从“谁态度不好”落到事实、时间和授权上。玩家要决定表格是保护事实，还是再次把人定罪。

现实触发：

- 软件工程课程分工表需要今晚确认。
- 班级材料里有一栏“本人确认”，不能由他人代填。
- 公共桌上放着周屿活动回执、唐骁评分细则、陆沉排班截图，三样东西都不能直接拿来当证据。

可拆节点：

| 节点 ID | 节点功能 | 场景 | 主要人物 |
|---|---|---|---|
| `DEFAULT-CH02-N01` | 决定表格格式 | 4XX 公共桌 | 林亦舟 / 唐骁 |
| `DEFAULT-CH02-N02` | 授权边界确认 | 公共桌 | 四人 |
| `DEFAULT-CH02-N03` | 第一次难听话 | 公共桌 / 阳台门口 | 周屿 / 唐骁 |
| `DEFAULT-CH02-N04` | 表格是否发群 | 微信群 | 四人 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `DEFAULT-CH02-C01` | 列事实，不写“责任人” | `dorm_trust +1`, `public_boundary +1` | 唐骁皱眉但接受 | 第五章项目协作更稳 |
| `DEFAULT-CH02-C02` | 按唐骁原表发群，先把事压住 | `project_stability +1`, `zhou_misread +1` | 周屿说“那我认罪行了吧” | 第六章复发强度上升 |
| `DEFAULT-CH02-C03` | 每个人自己补一行，不代填 | `repair_depth +1`, `time_debt +1` | 陆沉愿意坐下来打字 | 第七章毕业照授权更清楚 |
| `DEFAULT-CH02-C04` | 先问今晚各自必须赶什么 | `dorm_warmth +1`, `efficiency -1` | 唐骁提醒截止时间，周屿放下防御一点 | 第三章可开启多对象补话 |

内回流点：

- `RET-DEFAULT-05` 事项群短句。
- 回流方式一：表格发群后被唐骁用“截止 23:00”拉回，`rule_pressure +1`。
- 回流方式二：周屿只发一个表情，玩家可私聊或不追，分别影响 `zhou_trust` 与 `late_regret`。
- 消息回流：陆沉把“我自己填”发到群里又撤回，玩家如果只截图保存，`proxy_risk +1`；如果当面问，`luchen_trust +1`。

对话逻辑：

- 唐骁在意表格能不能解决事，但不是天然冷酷。
- 周屿讨厌“被写进表格”，尤其讨厌当众变成例子。
- 陆沉可以接受事实，不接受别人把他的沉默当同意。

防漂移：

- 表格不能成为万能解法。
- 不写“规则对、人情错”的单向判断。
- 不让林亦舟替陆沉完整表达结论。

### 第三章：私下补话

章节目标：让玩家承担“先找谁”的后果。修补不是平均分配温柔，补一个人就会制造另一个人的等待或误读。

现实触发：

- 公共桌谈完后，至少一人没有把话说完。
- 走廊、水房、食堂侧门、青枫居楼下都可能形成短谈话。
- 4XX 群里仍有未确认事项，玩家不能无限谈心。

可拆节点：

| 节点 ID | 节点功能 | 场景 | 主要人物 |
|---|---|---|---|
| `DEFAULT-CH03-N01` | 选择先补话对象 | 4XX 门口 | 林亦舟 |
| `DEFAULT-CH03-N02` | 周屿补话 | 食堂侧门 / 活动排练区边 | 林亦舟 / 周屿 |
| `DEFAULT-CH03-N03` | 唐骁补话 | C407 外 / 打印店 | 林亦舟 / 唐骁 |
| `DEFAULT-CH03-N04` | 陆沉补话 | 图书馆 / 阳光书屋外 | 林亦舟 / 陆沉 |
| `DEFAULT-CH03-N05` | 回到公共桌 | 4XX | 四人 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `DEFAULT-CH03-C01` | 先找周屿，问他为什么总把难堪讲成笑话 | `zhou_trust +1`, `tang_wait_cost +1` | 周屿一开始打岔，后来承认怕被当笑料 | 唐骁后续更强调截止 |
| `DEFAULT-CH03-C02` | 先找唐骁，问他为什么一定要表格先行 | `tang_trust +1`, `zhou_wait_cost +1` | 唐骁承认怕临时好说话变成默认责任 | 周屿后续更容易嘲“开会” |
| `DEFAULT-CH03-C03` | 先找陆沉，只问哪些话不能替他说 | `luchen_trust +1`, `proxy_risk -1` | 陆沉给一个具体边界，不给完整答案 | 第八章清寝旧物可降低代处理风险 |
| `DEFAULT-CH03-C04` | 谁都不找，等明天自然过去 | `repair_temperature -1`, `late_regret +1` | 表面平稳，群里语气更短 | 第六章复发时旧账更尖 |
| `DEFAULT-CH03-C05` | 三个人都发一句“明天找你单独说” | `time_debt +2`, `dorm_trust +1` | 每个人都知道自己不是唯一 | 后续章节增加迟到和精力代价 |

内回流点：

- `RET-DEFAULT-03` 凌空栈道软照面。
- 回流方式一：补话前走到凌空栈道，遇到夏知微，聊近况和未来，不聊冲突，`mind_clarity +1`，回公共桌后语气更稳。
- 回流方式二：聊到刚发生的事，夏知微提醒照片和材料都要授权，`public_boundary +1`，但 `time_debt +1`。
- 回流方式三：碰见许棠从融媒体方向出来，只问“材料本人确认了吗”，不展开她的主线，回到 4XX 时触发授权短句。

对话逻辑：

- 周屿补话要有抵抗，他会先反问“你现在代表谁”。
- 唐骁补话要有现实物件，例如表格、打印页、评分细则。
- 陆沉补话不能逼供，玩家追问过度会降低信任。

防漂移：

- 单独补话不能取代公共桌。
- 不能让玩家通过一次谈话同时安抚三个人。
- 夏知微、许棠只作为母版生活半径软照面，不开启新攻略线。

### 第四章：饭桌回声

章节目标：用吃饭和外卖检验宿舍温度。饭桌不是和解仪式，而是看还有没有人愿意替别人留一口、问一句、等十分钟。

现实触发：

- 群里只问“谁点饭”。
- 四人时间错开：周屿排练，唐骁赶项目，陆沉排班，林亦舟夹在中间。
- 可选地点包括西区外卖、食堂二楼、东区商业街、后街夜市。

可拆节点：

| 节点 ID | 节点功能 | 场景 | 主要人物 |
|---|---|---|---|

1. 周屿的台词要先绕、先玩笑、先护面子；真话只能在玩家不急着定罪时露出。
