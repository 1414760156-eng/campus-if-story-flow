# IF 分支开发文本写法规范

## 用途

本规范用于后续所有 `开发文本_IF分支*.md`。

这些文件不是母版小说正文，不是支线外传，不是试玩器，也不是直接给普通读者阅读的章节。它们是给后续互动剧情开发拆节点、选项、变量、事件回响和毕业结算用的方向稿。

母版小说负责判断人物性格、语气、关系逻辑、场景质感和对话风格。IF 开发文本只负责说明：这个分歧入口发生什么，玩家能怎么选，选择会把林亦舟推向哪种生活，谁的关系变强或变弱，后续卷如何承接。

## 输出结构

每条 IF 路线优先使用以下结构。

1. 路线定位：说明路线编号、入口时期、主角生活重心、核心代价、毕业结算温度。
2. 入口条件：说明触发事件、前置变量、互斥条件、是否能回主线。
3. 核心事件链：写 3-5 个关键事件，每个事件包含地点、参与人物、现实触发、玩家选择、结果变量、后续回响。
4. 可拆选择：每个关键事件至少提供 3 个可开发选项。选项不必都改变主线，但要改变心态、关系、资源、误读或后续文本。
5. 对话逻辑：不写完整对白，只说明人物在该节点会怎么说、为什么这么说、哪些话不能说。
6. 后续承接：说明第五卷、第六卷、第七卷分别如何承接，不提前替毕业结局写圆。

## 写法原则

- 直接写剧情方向和事件，不按母版小说那样长段铺叙。
- 只在必要处给出短示例句，用来判断语气，不写成长篇正文。
- 每个事件必须有现实压力：时间、钱、地点、手续、群消息、公开边界、寝室缺席、考试、项目、打工、情感误读。
- 选择必须有后果。后果可以很小，但不能消失。
- 被玩家靠近的人或方向获得更多剧情容量，未被选择的人仍按母版主线生活，不为了玩家重写人生。
- 晚风不是必选更进一步。她只是亲密外部方向的一种可能。
- 支线人物不是独立外传目标，而是生活半径、关系压力、价值对照和事件触发器。

## 禁止项

- 禁止把 IF 文本写成完整小说章节。
- 禁止写成“某人怕、某人怕、某人怕”式上帝视角总结。
- 禁止用“这一步会影响某某值”这种直接系统提示代替事件后果。
- 禁止用无功能环境开场，例如“第二天早上”“门缝里漏进来”“拖把水声”“阿姨喊”“睁眼”“手机震”等缓冲句。
- 禁止静态介绍桌面、穿着、外设、道具。物件只能通过动作、借用、遮掩、碰撞、争执、迟到代价、签字、取件、丢失等方式露出。
- 禁止把一条路线写成唯一正确答案。完美线、摆烂线、感情线、宿舍线、打工线都要有收益和代价。

## 推荐写法示例

### 错误方向

林亦舟站在走廊里，风从门缝吹进来。他忽然意识到，自己和宿舍的关系已经发生了变化。

### 正确方向

事件：主持招新复盘后，秦越临时把签到表和物资表交给林亦舟，要求今晚十点前发到群里。周屿还在台上补拍，唐骁在 4XX 群里催软件工程作业分工。

可选方向：
- 接下签到表，先把活动流程跑完：活动信用上升，4XX 作业确认延后。
- 只帮周屿核对名单，不接秦越的长期协助：周屿信任上升，活动线不锁定。
- 拒绝临时加活，回 4XX 处理作业：宿舍互信上升，完美线入口关闭。

对话逻辑：
- 秦越不煽情，只讲截止时间和责任归属。
- 周屿会用玩笑掩饰亏欠，但会记得林亦舟有没有留下。
- 唐骁不会指责，他只会把分工表先改完，这就是后续误读。

## 开发优先级

P0 阶段先写完整 IF 骨架：

- P0-A：第三幕社团 / 活动入口，形成完美线或自我管理线。
- P0-B：第四幕暑假 / 生活费 / 留校入口，形成兼职线或生存线。
- P0-C：第五幕前段亲密分歧入口，形成专注感情线。
- P0-D：第五幕中后段宿舍站队入口，形成周屿线、唐骁线、陆沉线、摆烂线。
- P0-E：第五幕隐藏高难入口，形成 5X 唯一硬外流线。

完成 P0 后，再开发 P1 支撑场景和 P2 碎片事件。
