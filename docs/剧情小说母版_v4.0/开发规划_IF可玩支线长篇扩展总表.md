# IF 可玩支线长篇扩展总表

## 用途

本文件用于把现有 P0-A 至 P0-E 的“节点骨架”升级为“可玩长支线开发标准”。

现阶段已有的 `开发规格_IF路线Schema与P0节点池.md` 负责定义字段、变量、节点和选项；本文件负责定义每条线扩展到什么体量、要有多少剧情密度、哪些事件必须可玩、哪些地方只能作为回声。

“符合母本长度”不理解为每条支线都复制整部母版的百万字体量，而是指：每条 IF 线必须拥有与其替代 / 扩展的母版段落相称的章节量、事件量、选择量和毕业后果，不能只用 3-5 个节点草草结算。玩家进入一条路线后，应感觉林亦舟真的把一段大学生活投向了另一种方向。

## 总原则

1. 主线锁点不变：军训手机事件、汇演、班会、主持招新、第五幕站队爆发、5X 唯一外流、第六卷关系定形、第七卷毕业收束仍发生。
2. IF 改变的是林亦舟的生活重心：靠近宿舍、活动、兼职、感情、某个室友、摆烂自保或 5X 硬外流。
3. 被选择的方向获得更长事件链、更高互动频率、更明确代价和更丰富毕业结算；未被选择的人仍按母版世界生活，只减少与林亦舟的交集。
4. 支线不是外传。它必须可玩：有入口、有选择、有回调、有资源代价、有关系变化、有阶段性结局。
5. 支线不是奖励线。每条线都必须同时给出吸引力与代价：时间、钱、精力、宿舍缺席、公开边界、误读、错过或低温结局。
6. 5X 仍是唯一硬外流。一旦进入，后续第六 / 第七卷只走 5X 单线；旧线角色可以同场擦肩，但不能开放回原多线的关键选择。

## 体量分级

| 等级 | 适用路线 | 建议章节 / 大段 | 建议字数 | 有效选择下限 | 事件密度 |
|---|---|---:|---:|---:|---|
| S 级主轴等量线 | `DEFAULT-4XX`、`R5X-HARD` | 10-12 章 | 80,000-120,000 字 | 35-50 次 | 每章 2-4 个选择，每 2-3 次选择产生回调 |
| A 级核心支线 | `R5-ROMANCE`、`R5-ZHOU`、`R5-TANG`、`R5-LUCHEN` | 8-10 章 | 60,000-90,000 字 | 28-40 次 | 每章至少 2 个选择，至少 3 次强后果回调 |
| B 级生活方式线 | `R3-PERFECT`、`R4-WORK`、`R5-LIEFLAT` | 6-8 章 | 45,000-70,000 字 | 20-30 次 | 每章至少 2 个选择，结尾必须形成明确生活方式后果 |

说明：

- “有效选择”指点击后至少改变关系、变量、文本回声、资源状态或结局温度，不能只是翻页按钮。
- 完美线和摆烂线可以弱冲突，但不能无后果。完美线的代价是低温、延迟和被工具化；摆烂线的代价是入口过期、被少问一次和毕业时低存在感。
- 亲密线不能写成无成本恋爱奖励。甜、钱、时间、宿舍缺席和公开误读必须同场出现。
- 宿舍三室友线不能写成“选谁谁被拯救”。靠近某个人会提高一部分信任，也会制造另外两个人的误读和距离。

## 每条可玩支线的交付结构

每条长支线单独建立一份开发文件，建议文件名：

- `开发长线_IF可玩支线DEFAULT-4XX宿舍修补线.md`
- `开发长线_IF可玩支线R3-PERFECT社团完美线.md`
- `开发长线_IF可玩支线R4-WORK暑假兼职线.md`
- `开发长线_IF可玩支线R5-ROMANCE专注感情线.md`
- `开发长线_IF可玩支线R5-ZHOU周屿线.md`
- `开发长线_IF可玩支线R5-TANG唐骁线.md`
- `开发长线_IF可玩支线R5-LUCHEN陆沉线.md`
- `开发长线_IF可玩支线R5-LIEFLAT摆烂线.md`
- `开发长线_IF可玩支线R5X-HARD唯一硬外流线.md`

每个文件固定包含：

1. 路线定位：这条线让林亦舟把生活投向哪里。
2. 进入条件：由哪些时期外流点、变量和前置选择触发。
3. 不改变的母本锁点：哪些事仍发生，只改变林亦舟的位置和关系温度。
4. 章节 / 大段表：每章的事件、地点、参与人物、冲突、选择、回调。
5. 选择节点表：每个节点至少 2-4 个可选动作，标明变量变化。
6. 资源与代价：时间、钱、精力、公开边界、宿舍缺席、错过入口。
7. 对话逻辑：写人物会怎么说，不必写完整小说对白。
8. 毕业结算：第七卷如何形成说开、淡化、体面距离、不和解、继续联系或遗憾。
9. 防漂移线：明确这条线不能写成什么。

## 路线扩展总览

### DEFAULT-4XX：宿舍修补线

等级：S 级。

核心体验：玩家选择留在 4XX 的旧账里，不逃到外部关系，也不只站某一个人。可玩重点不是圆满，而是一次次决定“现在是把事实说清、先做事、还是承认自己想逃”。

章节方向：

1. 事项群变短：群消息从聊天变成短句，玩家决定当面说还是继续用表格。
2. 公共桌事实表：列事实、删定罪词、承认自己缺席或先问今晚必须做什么。
3. 私下补话：选择先补周屿、唐骁、陆沉，或谁都不找。
4. 饭桌回声：一次饭局不是和解，只是看谁还愿意坐下。
5. 项目 / 材料协作：专业配合恢复，但亲密不自动恢复。
6. 复发：旧账被新事再次拉出来，玩家决定是否承认前一次只是暂时压住。
7. 毕业照：同框资格、迟到、不补拍和“仅留存”边界。
8. 清寝：公共物、快递、遥控器、旧 U 盘和谁有权处理。
9. 最后一顿饭：到场、迟到、错过和不替别人确认。
10. 武生院站：说开一点、淡化一点、体面但不亲近或不和解。

最低选择密度：40 次左右。

主要变量：`dorm_trust`、`dorm_warmth`、`avoidance`、`old_debt`、`repair_depth`、`graduation_temperature`。

防漂移：不能写成四人最终大团圆；修补只增加当面处理机会，不保证结局圆满。

### R3-PERFECT：第三幕社团 / 完美线

等级：B 级。

核心体验：玩家让林亦舟更早进入活动、主持、材料和公开表达的秩序里。他会越来越会做事，也越来越容易被别人当成“可靠工具”。

章节方向：

1. 主持招新协助表：接流程、帮周屿、回 4XX 或补秦越漏洞。
2. 班会材料归档：授权边界、找沈嘉禾核格式、找唐骁拆时间。
3. 公开表达低风险化：上台、串稿、采访、删私人信息。
4. 活动信用积累：别人开始默认找他兜底。
5. 宿舍退温：4XX 发现他总能把事情处理好，却不一定还在场。
6. 第五幕回声：站队爆发时，他更擅长写表，却更难直接说难听话。
7. 第六 / 七卷结算：公开材料稳定，私人关系低温。

最低选择密度：20-28 次。

主要变量：`self_control`、`public_credit`、`activity_link`、`time_debt`、`dorm_warmth`、`public_boundary`。

防漂移：不能写成奖状线或正确线。越完美，越要付出亲近延迟和被工具化代价。

### R4-WORK：第四幕暑假 / 兼职线

等级：B 级。

核心体验：玩家让林亦舟把暑假投进钱、排班、生活费和现实压力里。他不是变励志，而是慢慢学会把生活拆成班次、路费、饭钱和能不能赴约。

章节方向：

1. 生活费延迟：问父母、压支出、接临时跑腿或找陆沉问勤工。
2. 快递站排班：半天班、迟到饭局、换班和被默认缺席。
3. 打印店证明材料：勤工表、课程材料、家庭暗线和格式边界。
4. 错过晚风 / 活动圈：不是不想去，而是排班、钱和疲惫挤掉入口。
5. 宿舍默认不等：周屿少叫一次饭，唐骁排表加限制，陆沉理解但不代言。
6. 第五幕回声：站队爆发时，林亦舟更现实，也更容易被说“不在现场”。
7. 第六 / 七卷结算：工作能力增强，但许多关系入口过期。

最低选择密度：22-30 次。

主要变量：`money_pressure`、`work_shift`、`family_signal`、`time_debt`、`dorm_absence`、`missed_chance`。

防漂移：不能写成苦情线或励志线。核心是钱和时间如何改写生活半径。

### R5-ROMANCE：第五幕亲密分歧 / 专注感情线

等级：A 级。

核心体验：玩家不默认和晚风更进一步，而是在第五幕前段选择把情绪和时间交给沈嘉禾或夏知微。关系变甜，但也让 4XX 的缺席、误读和错过变得更明显。

候选方向：

- 沈嘉禾：现实协作、资料、打印、边界、未来选择。
- 夏知微：影像、观看、授权、表达、毕业照片。
- 晚风可保留为朋友、游戏搭子、错过或低频生活回声，不自动失败。

章节方向：

1. 晚风停在门外：说清不进一步、保持朋友、逃回 4XX 或含糊拖延。
2. 候选确认：沈嘉禾 / 夏知微方向成立，或亲密线不成型。
3. 第一次现实陪伴增频：聊宿舍、聊近况、聊未来，或只帮一件事。
4. 礼物 / 节日 / 笨拙模仿：甜必须同时有钱、时间和误读代价。
5. 公开边界：朋友圈、照片、资料、署名和“要不要被看见”。
6. 宿舍缺席后果：4XX 不是阻碍恋爱，而是没人再默认他在场。
7. 冲突反照：外部关系中的幼稚、吃味、AA、礼物账单和公开压力。
8. 第六卷关系成形：选择亲密对象的生活圈成为主叙事半径。
9. 第七卷毕业结算：继续联系、异地淡化、说清但不承诺或毕业后错过。

最低选择密度：32-40 次。

主要变量：`romance_focus`、`romance_candidate`、`public_boundary`、`gift_debt`、`money_pressure`、`time_debt`、`dorm_visibility`。

防漂移：不能写成“选女主奖励”。亲密越近，越要写清代价、边界和缺席后果。

### R5-ZHOU：周屿线

等级：A 级。

核心体验：玩家靠近周屿，进入热闹、主持、面子和玩笑背后的不安。可玩重点是“帮他圆场”还是“当场截住”，以及是否愿意承认热闹也会伤人。

章节方向：

1. 玩笑挡不住：帮圆场、当场截住、私下提醒或不接。
2. 活动圈站边：去补场、拉回 4XX、找秦越流程解决。
3. 面子债：请客式补偿、拒绝热闹包装、只吃饭不谈。
4. 误读扩大：唐骁觉得他偏周屿，陆沉觉得他又替别人说完。
5. 主持 / 小舞台压力：公开表达和私人关系互相挤压。
6. 周屿第一次说出难听话：玩家选择护面子、追事实或先停场。
7. 第六卷窄化：活动圈关系更强，4XX 专业协作变低温。
8. 第七卷结算：半句道歉、热闹淡化、继续联系或体面远离。

最低选择密度：28-36 次。

主要变量：`zhou_trust`、`activity_link`、`public_smooth`、`dorm_absence`、`money_boundary`、`late_regret`。

防漂移：不能写成单纯帮朋友。周屿线必须有面子、玩笑、公开热闹和私下受伤的双面性。

### R5-TANG：唐骁线

等级：A 级。

核心体验：玩家靠近唐骁，进入规则、项目、流程和公平的压力里。可玩重点是规则如何保护人，也如何把人推远。

章节方向：

1. 时间线不是判决书：核事实、删定罪词、按原表发群或先问授权。
2. 项目优先：先稳 C407、强行拉回宿舍或找老师确认边界。
3. 公平的代价：按规则执行、拆最低责任、让唐骁自己说。
4. 专业配合但不亲密：接口、归档包、说明页和演示口径各守一栏。
5. 被误读成审判：周屿觉得没面子，陆沉觉得自己被表格代替。
6. 唐骁的裂口：他承认自己怕流程漏洞，也怕别人把错推给他。
7. 第六卷窄化：项目稳定提高，宿舍温度下降。
8. 第七卷结算：实用照看、体面距离、继续协作或不再私聊。

最低选择密度：28-36 次。

主要变量：`tang_trust`、`project_stability`、`public_boundary`、`rule_pressure`、`repair_depth`、`dorm_warmth`。

防漂移：不能写成“唐骁永远正确”。规则既是保护，也是伤人的工具。

### R5-LUCHEN：陆沉线

等级：A 级。

核心体验：玩家靠近陆沉，但不能拯救他，不能替他说。可玩重点是“给他留入口”而不是“帮他说完整答案”。

章节方向：

1. 不替他说：只说事实、不说情绪、把问题交回陆沉。
2. 现实压力撞上宿舍规则：排班、家里消息、勤工和证书材料。
3. 沉默不是答案：等他说、问最低可公开事实、挡掉追问。
4. 图书馆 / 书屋 / 快递站生活半径：陆沉的稳定不是冷淡。
5. 其他人误读：周屿觉得扫兴，唐骁觉得边界不清。
6. 陆沉多说一句：玩家选择接住、追问、替解释或停止。
7. 第六卷窄化：现实压力被看见，但仍不变成拯救线。
8. 第七卷结算：旧书、补登记、迟来的解释、继续低频联系或淡化。

最低选择密度：28-36 次。

主要变量：`luchen_trust`、`proxy_risk`、`work_reality`、`family_pressure`、`quiet_bond`、`repair_temperature`。

防漂移：不能写成救赎线。林亦舟只能降低替他说话的风险，不能把陆沉改写成外向的人。

### R5-LIEFLAT：摆烂 / 不站队线

等级：B 级。

核心体验：玩家不站队，不处理，少回几条消息，错过几个入口。它不大爆炸，却让所有人慢慢不再等他。

章节方向：

1. 不回群：静音、只发表情、刷手机、去买吃的。
2. 弱冲突累积：不叫人上课、不帮取外卖、不解释缺席。
3. 游戏 / 外卖 / 夜市短逃避：轻松是真的，但不能抵消入口过期。
4. 关系低温：别人不骂他，只是不再问他。
5. 入口过期：想补时饭局空栏、材料已交、照片已拍。
6. 第六卷窄化：4XX 仍存在，但他越来越像旁观者。
7. 第七卷结算：体面淡化、错过、群不解散但没人单独找他。

最低选择密度：20-28 次。

主要变量：`avoidance`、`lieflat_score`、`low_conflict`、`missed_chance`、`dorm_trust`、`late_regret`。

防漂移：不能写成搞笑线或纯惩罚线。摆烂的真实感来自“小事不处理导致入口过期”。

### R5X-HARD：第五幕隐藏高难 / 唯一硬外流线

等级：S 级。

核心体验：玩家在第五幕唯一硬外流点选择彻底离开原有多线，进入许澈、韩野、蒋沐的新圈子。新圈轻松、无旧账、无需解释，但也薄、散、不承担深责任。

章节方向：

1. 最后一扇门：回头敲门、下楼冷静、跟许澈走或静音群继续离开。
2. 新桌子：新圈座位、饮料、玩笑和没有旧账的轻松。
3. 薄关系显形：补钱、问清、想回头、装没事。
4. 新圈生活半径：后街、东区商业街、校外短行程、游戏局和临时聚餐。
5. 旧人同场擦肩：点头、只读、短句问候、继续新线。
6. 第六卷单线：新圈占据主叙事，4XX 只作为同场回声。
7. 第七卷毕业：新人散得轻，旧人远得重。
8. 5X 结算：发旧消息、对许澈说清、删置顶或什么都不做。

最低选择密度：40-50 次。

主要变量：`route_lock`、`hard_outflow`、`new_circle_trust`、`thin_relation`、`old_dorm_distance`、`soft_passing`、`5x_regret`。

防漂移：不能写成“新人更好”。5X 的宿命感来自玩家终于逃离旧账，却发现新关系不一定愿意替他承担重量。

## 可玩性最低配置

每条路线正式扩写时必须满足：

1. 至少 3 个稳定场景反复出现，并随关系变化改变功能。
2. 至少 3 个命名人物或稳定生活人物参与路线，不做一次性 NPC。
3. 每章至少 1 个现实任务：签到、表格、排班、材料、饭局、礼物、考试、照片、清寝、离校手续等。
4. 每章至少 1 个关系压力：谁等谁、谁不等谁、谁替谁说话、谁不再问、谁被公开、谁缺席。
5. 每 2-3 个选择必须有一次回调，不能只在结局结算。
6. 结局必须有多温度：说开、淡化、体面距离、不和解、继续联系、错过、5X 遗憾。
7. 支线里的环境描写只为阻碍、关系、后果或选择服务，不写无功能开场。

## 推荐开发顺序
