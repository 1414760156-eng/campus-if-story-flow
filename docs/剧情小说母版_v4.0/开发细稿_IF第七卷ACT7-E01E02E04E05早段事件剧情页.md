# IF 第七卷 ACT7-E01 / E02 / E04 / E05 早段事件剧情页级细稿

## 用途

本文用于把 DEFAULT-4XX 第七卷 4 个早段事件拆成可开发剧情页、选项页、变量回调和 JSON 事件结构；本细稿已用于正式 JSON 的 12 事件闭环接入。

本文件不是小说正文，也不是试玩器脚本。它服务于下一步补齐 `开发数据_IF剧情页级JSON_DEFAULT-4XX_v1.json` 中尚缺的第七卷事件：

- `ACT7-E01-PRE-REGISTER`
- `ACT7-E02-PUBLIC-SCOPE`
- `ACT7-E04-THESIS-TOPIC`
- `ACT7-E05-MARCH-QUEUE`

本轮范围只覆盖：

| 字段 | 值 |
|---|---|
| `route_id` | `DEFAULT-4XX` |
| `route_pool_id` | `POOL-DEFAULT-4XX` |
| `act7_variant_id` | `ACT7-DEFAULT` |
| `route_switch_allowed` | `false` |
| `source_mother_volume` | 第七卷《最后一个新年》 |

## 总体承接

E01、E02、E04、E05 是 DEFAULT-4XX 第七卷的“早段压力链”：先确认毕业后能不能找到人，再确认什么能公开，再确认毕业设计方向和导师签字，最后把三四月的实习、复试、证书、开题复核和毕业照信息采集压到同一时间段。

这 4 个事件的功能不是另开路线，而是给后续 E03、E06、E07、E08-E12 提供硬前置字段。

## 后续 JSON 接入建议

正式接入 `开发数据_IF剧情页级JSON_DEFAULT-4XX_v1.json` 时，以上 4 个对象应补进 `act7_settlement_events`，并把 `validation_targets.act7_core_event_count` 从 8 更新为 12；当前 DEFAULT-4XX 正式 JSON 已完成该接入。

接入后，DEFAULT-4XX 第七卷正式 JSON 应覆盖完整 12 事件：

```json
[
  "ACT7-E01-PRE-REGISTER",
  "ACT7-E02-PUBLIC-SCOPE",
  "ACT7-E03-LAST-NEW-YEAR",
  "ACT7-E04-THESIS-TOPIC",
  "ACT7-E05-MARCH-QUEUE",
  "ACT7-E06-THESIS-REVISION",
  "ACT7-E07-DEFENSE",
  "ACT7-E08-GRAD-PHOTO",
  "ACT7-E09-DORM-CLEAR",
  "ACT7-E10-LAST-MEAL",
  "ACT7-E11-LEAVING-PAPERS",
  "ACT7-E12-STATION-AFTER"
]
```

校验时至少检查：

- 12 个事件都在同一 `route_pool_id = POOL-DEFAULT-4XX` 下。
- 每个事件都有 4 个剧情页、1 个二方向选择窗口。
- 每个事件都保持 `route_switch_allowed = false`。
- `ACT7-E01` 至 `ACT7-E05` 能顺序承接到 E06，不出现跳线或重开路线。
