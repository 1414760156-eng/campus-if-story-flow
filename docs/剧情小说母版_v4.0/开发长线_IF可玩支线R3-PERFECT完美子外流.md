# 开发长线_IF可玩支线R3-PERFECT完美子外流

## 用途

本文用于把 `R3-PERFECT` 从 `A3-ACTIVITY-PUBLIC` 父池下的 `mode_perfect` 子外流扩展成可拆剧情页、选项页、变量回调和毕业结算的 B 级生活方式线。

它不是 A3 父池本体，也不是 `R5-TANG` 唐骁规则线。它规定的是：玩家在第三幕父池里持续选择流程、低风险、按时、公开可控和自我管理后，林亦舟如何被活动圈、老师、材料窗口和同学们记成“可靠的人”，又如何因此把私人解释、宿舍温度和亲密关系推迟到更后面。

本线重点不是“优秀奖励”，而是公开信用的代价。林亦舟越会把事情处理好，越容易被继续加活；越会写清边界，越容易把难听的话留到下一次；越显得没出错，越少有人问他是不是也被事情压住。

## 路线定位

路线编号：`R3-PERFECT`

路线池编号：`POOL-R3-PERFECT`

父池编号：`POOL-A3-ACTIVITY-PUBLIC`

路线等级：B 级生活方式子外流线

入口时期：第三幕 `A3-ACTIVITY-PUBLIC` 父池第三次收束后。

共享范围：第一卷、第二卷共享；第三幕从 A3 父池派生；第四幕、第五幕、第六幕和第七幕仍嫁接母本公共锁点，但林亦舟进入公共事件的姿势变成流程、自控、公开信用和边界说明。

核心体验：玩家让林亦舟成为“能处理事的人”。他会更早学会写停止条件、删私人信息、保留授权记录、把活动风险压到最低；但他也会更习惯把情绪延后，把关系变成事项，把求助变成“我先处理完”。玩家要处理的不是“要不要变优秀”，而是：当一个人总能把事情做对，别人是否还会记得他也需要被问一句。

## 进入条件

必须先满足 A3 父池收束：

| 条件 | 建议值 | 说明 |
|---|---:|---|
| `route_parent_pool_id` | `POOL-A3-ACTIVITY-PUBLIC` | 必须来自第三幕社团 / 活动父池 |
| `a3_final_choice` | `keep_first` 或 `keep_second` | 必须坚持单一方向，不能 `take_both` 或 `take_neither` |
| `route_mode_tag` | `mode_perfect` | 流程、低风险、按时、公开可控持续成立 |
| `self_control` | `>= 4` | 多次选择自我管理和流程优先 |
| `public_credit` | `>= 3` | 活动圈或老师已经把林亦舟记成可靠的人 |
| `emotional_delay` | `>= 2` | 多次先处理事务、延后解释关系 |
| `dorm_warmth` | `<= 1` | 4XX 日常温度已经因缺席或事项化下降 |

互斥规则：

- `a3_final_choice = take_both` 时进入压力模式，不得锁 `R3-PERFECT`。
- `a3_final_choice = take_neither` 时回宿舍或回避倾向，不得锁 `R3-PERFECT`。
- 主要跟随唐骁处理第五幕站队后的规则和项目责任，属于 `R5-TANG`，不是本线。
- 主要跟随沈嘉禾或夏知微发展亲密投入，且候选来源合法，属于 `R5-ROMANCE`，不是本线。
- 只参加一次社团、只做一次材料、只被夸一次靠谱，不足以锁本线。

## 不改变的母本锁点

- 汇演、班会、主持招新、照片授权和活动材料仍按母本发生。
- 4XX 的旧账不会因为林亦舟公开信用变高而消失。
- 第四幕暑假、生活费和家庭暗线仍发生；本线不能取消 `R4-WORK` 的现实压力，只能改变林亦舟处理压力的方式。
- 第五幕宿舍强冲突仍发生；完美线只让林亦舟更倾向于损失控制，不能取消站队后果。
- 唐骁仍有自己的规则和项目线；本线中的唐骁只能作为流程镜像或协作压力，不能抢成 `R5-TANG`。
- 沈嘉禾和夏知微仍按 A3 候选权限存在；本线不能把她们自动写成亲密对象。
- 第六卷 C407、材料归档、公开边界和关系定形仍发生；只是林亦舟更多从“先把风险降下来”进入。
- 第七卷预登记、毕业照、清寝、饭局和离校手续仍发生。
- 5X 仍是唯一硬外流。本线不能把公开圈层写成无旧账新世界。

## 第五 / 第六卷硬约束接入

`R3-PERFECT` 起源于第三幕，但普通长线完整锁定前仍必须服从第五卷最低主线选择要求：

- 至少 5 个主线选择窗口：宿舍站队、亲密 / 晚风、项目 / 规则、活动 / 兴趣、回避 / 外流。
- 每个计入窗口都必须有 `act6_echo_hook`。
- 未达标时，`R3-PERFECT` 只能作为公开信用倾向或低温软回声，不能直接锁完整第六卷。
- 第六卷进入 `POOL-R3-PERFECT` 后，固定 20 个剧情段、5 个四段组、5 个二方向选择窗口；每个窗口只给两个当前池内方向。

## 核心变量

| 变量 | 含义 | 高值倾向 | 风险 |
|---|---|---|---|
| `self_control` | 自我管理、按时、流程优先 | 能把混乱事件压成可执行事项 | 情绪表达减少 |
| `public_credit` | 老师、活动圈、材料窗口眼中的可靠度 | 更常被继续交付任务 | 被默认兜底 |
| `scope_clarity` | 职责范围和停止条件是否写清 | 不乱替人背责任 | 过高时显得冷、拒人 |
| `tooling_risk` | 被当成可靠工具的风险 | 别人默认他能处理 | 私人需求被忽略 |
| `emotional_delay` | 把关系问题推迟处理 | 先交材料、先保现场 | 第五卷旧账加重 |
| `private_delay` | 私人解释延后程度 | 很少当场说难听话 | 亲近被延迟到过期 |
| `activity_link` | 活动圈牵引 | 秦越、许棠、老师、摄影社等更常找他 | 宿舍误读他“不在” |
| `public_boundary` | 采访、照片、署名、截图、授权边界 | 公开风险更低 | 低值引发授权或留证问题 |
| `dorm_warmth` | 4XX 日常温度 | 仍有饭局和当面解释入口 | 高自控会持续降温 |
| `public_misread` | 别人对他“没情绪”的误读 | 被认为冷静、会处理、不会受伤 | 很少有人问他的感受 |
| `graduation_temperature` | 毕业结算温度 | 清楚、体面、可联系、低温或错过 | 综合前述变量决定 |

共享回声变量：

| 变量 | 来源 | 在本线中的用途 |
|---|---|---|
| `project_stability` | C407 / 项目协作 | 只作为公开流程和材料稳定回声，不转成唐骁线。 |
| `shen_jiahe_link` | A3 材料协作 | 只作为材料边界或现实协作参照，不自动开感情线。 |
| `xia_zhiwei_link` | A3 摄影 / 授权 | 只作为公开观看和照片边界参照，不自动开感情线。 |
| `zhou_trust` | 主持 / 活动圈 | 标记周屿是否觉得林亦舟进入了活动场，但不直接锁周屿线。 |
| `old_debt` | 4XX 旧账 | 标记未解释的私人问题是否拖到第五 / 第六卷。 |

说明：上述共享回声变量只用于 cross_route_echo；本线核心路由仍由 `POOL-R3-PERFECT` 的公开信用、自控、被工具化和低温变量控制。

## 体量与密度

建议 8 章开发体量，对应约 50,000-75,000 字正文容量。本文只提供可开发骨架，不直接写足正文。

最低配置：

- 至少 28 次有效选择。
- 至少 6 个内部回流点。
- 每个内部回流点至少 2 个方向，建议 3 个方向。
- 每个回流方向至少 3-4 个连续剧情拍点。
- 至少 3 次强后果回调：活动负责人默认找他、4XX 事项群冷化、公开边界处理影响第五卷、第七卷毕业材料体面但私人低温。

## 章节开发详表

### 第一章：停止条件写在最后一行

章节目标：从 A3 父池第三次收束进入 `R3-PERFECT`。林亦舟不是“加入社团”，而是接下固定职责，同时把停止条件写清。

现实触发：

- 秦越或活动负责人问他能不能固定对表。
- 老师希望班会材料归档以后继续由他确认格式。
- 4XX 群里有人问今晚谁回去带钥匙。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |
|---|---|---|---|
| `PERFECT-CH01-N01` | 子外流入口核验 | 明德楼走廊 / 招新桌 | 林亦舟、秦越 |
| `PERFECT-CH01-N02` | 写职责范围 | 活动群公告 / 共享文档 | 林亦舟、许棠 |
| `PERFECT-CH01-N03` | 宿舍第一次退温 | 青枫居4XX | 周屿、唐骁、陆沉 |
| `PERFECT-CH01-N04` | 入口确认 | 表单提交页 | 林亦舟 |

可拆选择：

| 选择 ID | 选择 | 变量变化 | 后果 |
|---|---|---|---|
| `PERFECT-CH01-C01` | 接固定流程，但写清职责范围 | `self_control +1`、`scope_clarity +1`、`public_credit +1` | 锁定 `R3-PERFECT` 倾向 |
| `PERFECT-CH01-C02` | 接下流程，不写停止条件 | `public_credit +1`、`tooling_risk +1` | 后续容易被继续加活 |
| `PERFECT-CH01-C03` | 拒绝固定职责，回 4XX 当面解释 | `dorm_warmth +1`、`public_credit -1` | 退出本线，回宿舍主轴 |
| `PERFECT-CH01-C04` | 接固定职责，同时替周屿兜活动情绪 | `zhou_trust +1`、`tooling_risk +1` | 转周屿线前置风险，本线稳定度下降 |

章节钩子：林亦舟在共享文档最后一行写“本表只负责现场流程，不负责私人解释”，周屿看见后笑了一下，说：“你现在连自己都能写进备注里了。”

### 第二章：被转发的签到表

章节目标：公开信用开始生效。一个表被转发，林亦舟从“帮一次忙的人”变成“下次找他的人”。

现实触发：

- 招新签到表被转发到活动负责人群。
- 秦越直接把候场顺序发给他。
- 唐骁在 4XX 群里按“林亦舟不在”重新排作业讨论。

可拆选择：

| 选择 ID | 选择 | 变量变化 | 后果 |
|---|---|---|---|
| `PERFECT-CH02-C01` | 继续对表，先保现场不乱 | `public_credit +1`、`activity_link +1`、`time_debt +1` | 秦越以后默认找他 |
| `PERFECT-CH02-C02` | 要求表单写负责人，不让自己口头兜底 | `scope_clarity +1`、`public_boundary +1` | 公开边界清楚，但显得不够热络 |
| `PERFECT-CH02-C03` | 把表转回负责人，自己只做提醒 | `self_control +1`、`tooling_risk -1` | 可靠但不全接 |
| `PERFECT-CH02-C04` | 为了不扫兴，临时接下全部候场 | `tooling_risk +2`、`dorm_warmth -1` | 宿舍错过加深 |

对话逻辑：

- 秦越只问能不能做，不用热情绑架他。
- 周屿会说“你现在比主持队还像主持队的人”。
- 唐骁不会责怪，只会把讨论时间按现实重新排。
- 陆沉不多评价，但会把公共桌上的材料往旁边挪一点。

章节钩子：活动结束后，群里有人发“今天流程顺多了，下次还是林亦舟对表吧”。这句话下面没有人反对。

### 第三章：今晚能交最好

章节目标：班会材料、照片授权和窗口截止把本线从活动场带到公开材料。林亦舟越能处理风险，越容易把私人解释往后放。

现实触发：

- 辅导员说班会材料“今晚能交最好”。
- 许棠问公开稿能不能写寝室号。
- 沈嘉禾提醒附件顺序不对，但不替他背责任。
- 4XX 晚饭也在同一时间。

可拆选择：

| 选择 ID | 选择 | 变量变化 | 后果 |
|---|---|---|---|
| `PERFECT-CH03-C01` | 先删寝室号和私人细节，再交公开稿 | `public_boundary +2`、`public_credit +1` | 公开风险降低 |
| `PERFECT-CH03-C02` | 找沈嘉禾只核格式，写清不让她替写 | `shen_jiahe_link +1`、`scope_clarity +1` | 保留材料协作软回声，不开感情线 |
| `PERFECT-CH03-C03` | 先回 4XX 吃饭，材料明早补交 | `dorm_warmth +1`、`public_credit -1` | 完美线降温，可回宿舍 |
| `PERFECT-CH03-C04` | 做完整归档，晚饭只在群里说一句 | `public_credit +1`、`private_delay +1`、`dorm_warmth -1` | 私人解释延后 |

对话逻辑：

- 许棠关心能不能公开，不替他写漂亮说法。
- 沈嘉禾会提醒“停止条件也要写进文档，不然你以后很难退”。
- 唐骁会认可材料清楚，但不会因此替他说情绪。

章节钩子：林亦舟把“今晚能交最好”改成“今晚 22:00 前提交初版”，按下保存时，4XX 群里的晚饭照片已经刷到第二张。

### 第四章：不上台的人也被点名

章节目标：公开表达低风险化。林亦舟可以不上台、只做幕后，但公开信用仍会把他推到被点名的位置。

现实触发：

- 汇演复盘会需要串稿和采访材料。
- 夏知微问是否能拍正脸或背影。
- 老师在复盘里点名“流程表做得清楚”。

可拆选择：

| 选择 ID | 选择 | 变量变化 | 后果 |
|---|---|---|---|
| `PERFECT-CH04-C01` | 做幕后串稿，不上台 | `self_control +1`、`public_credit +1` | 被认可但不一定被记住本人 |
| `PERFECT-CH04-C02` | 上台做简短说明，只说流程不说私人 | `public_credit +2`、`public_misread +1` | 别人以为他冷静 |
| `PERFECT-CH04-C03` | 拒绝正脸，只允许拍手稿和背影 | `public_boundary +1`、`xia_zhiwei_link +1` | 夏知微软回声，不开感情线 |
| `PERFECT-CH04-C04` | 接受采访，但要求删私人细节 | `public_boundary +2`、`scope_clarity +1` | 公开叙事更稳 |

章节钩子：掌声落下时，周屿站在舞台侧边看他，忽然说：“你不上台也能被表扬，挺厉害的。”语气很轻，不像夸奖。

### 第五章：可靠不是亲近

章节目标：把完美线的代价落回 4XX。室友会承认他靠谱，但更少私下找他。

现实触发：

- 公共桌上堆着洗衣卡、外卖小票、取件码和未签字表。
- 唐骁认可清单，周屿嫌寝室像办公室，陆沉把自己的东西移出公共区。
- 林亦舟很快能处理完，却不知道处理完以后要不要坐下来。

可拆选择：

| 选择 ID | 选择 | 变量变化 | 后果 |
|---|---|---|---|
| `PERFECT-CH05-C01` | 把公共桌事项列成清单，谁的东西谁确认 | `self_control +1`、`dorm_warmth -1`、`public_misread +1` | 室友觉得省事也觉得冷 |
| `PERFECT-CH05-C02` | 只提醒截止，不追问原因 | `emotional_delay +1`、`old_debt +1` | 第五卷旧账加深 |
