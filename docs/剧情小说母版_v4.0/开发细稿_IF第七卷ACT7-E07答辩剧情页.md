# IF 第七卷 ACT7-E07 答辩 / 答辩后剧情页级细稿

## 用途

本文用于把 `ACT7-E07-DEFENSE` 拆成可开发剧情页、选项页、变量回调和后续 JSON 事件结构。

本文件不是小说正文，也不是试玩器脚本。它服务于第七卷毕业结算层：同一个“答辩 / 答辩后”事件点，所有已锁定路线都必须经过，但每条路线读取自己的 `route_pool_id`、`act7_variant_id`、`act6_final_direction`、`graduation_temperature`，并继承 `ACT7-E06-THESIS-REVISION` 的提交版本、帮看关系和边界检查结果，输出不同的答辩门口、补交顺序、消息回声和毕业温度。

## 接入位置

| 字段 | 建议值 |
|---|---|
| `event_id` | `ACT7-E07-DEFENSE` |
| `act` | `act7` |
| `mother_chapter` | 第七卷第七章《答辩以后》 |
| `previous_event` | `ACT7-E06-THESIS-REVISION` |
| `next_event` | `ACT7-E08-GRAD-PHOTO` |
| `route_switch_allowed` | `false` |
| `same_mother_event` | `true` |
| `main_location` | 教学楼 C407、答辩教室走廊、打印店、B204、共享盘归档窗口 |
| `base_scene` | 答辩顺序表、最终版修改、签字、小章、修改后提交、答辩后第一条消息 |

本事件必须承接 `ACT7-E06-THESIS-REVISION`：上一事件决定了修改稿版本、谁帮看、谁没等、课程成果和主题材料边界是否分清；本事件把这些结果推进到答辩门口、答辩后补交和“谁还在不在场”的毕业回声。

它不能写成“答辩通过了”一句话。答辩的功能是检验当前路线关系是否还能等到门口、是否还能处理补交、是否还能在通过以后说一句不服务流程的话。

## 共用硬规则

1. 每条路线必须读取 `ACT7-E06` 的至少三个字段：`thesis_revision_version`、`thesis_revision_helper`、`thesis_boundary_checked`。建议同时读取 `revision_submitted_before_deadline`。
2. 每条路线必须出现答辩顺序表、答辩门口或走廊等待、答辩后修改 / 签字 / 小章 / 补交中的至少三项。
3. 每条路线必须让玩家做一次核心二方向选择：等 / 不等，先补材料 / 先回消息，让别人等 / 让别人走，认真说结果 / 用玩笑糊过去。二方向按路线语境改名，但不能扩成新路线。
4. 每条路线必须写清“在门口”不等于“关系恢复”。有人等、没人等、只发消息、赶流程、迟到或只在同场擦肩，都可以成立。
5. 答辩后补交必须带现实动作：签字、盖章、小章、打印、扫描、上传、共享盘缓存或修改后提交。不能只写“老师说可以了”。
6. `R5X-HARD` 仍保持 `route_lock = "5x"` 和 `hard_outflow = true`，旧线人物只能软照面、消息、同场流程或低温回声，不得回原多线。

禁止写法：

- 不允许把答辩门口写成所有人等他并自然和好。
- 不允许用“通过了”一次性清掉第五卷 / 第六卷旧账。
- 不允许因为某人在答辩后等他，就重新打开另一条完整路线。
- 不允许让晚风、沈嘉禾、夏知微、许澈或新圈人物替他承担答辩材料责任。
- 不允许把答辩写成纯流程通关；每个流程动作都必须带出关系后果。

## 从修改稿到答辩后

| 承接项 | E06 修改阶段 | E07 答辩阶段 | 可回调变量 |
|---|---|---|---|
| 版本 | 修改稿、查重前版、终版 | 答辩用版、修改后提交版、归档版 | `thesis_revision_version`、`defense_version_locked` |
| 帮看 | 谁看格式、来源、授权或摘要 | 谁在门口、谁只发消息、谁不再等 | `thesis_revision_helper`、`defense_waiting_state` |
| 边界 | 课程成果、主题材料、私人故事分清 | 答辩后是否误把私人故事写入公开材料 | `thesis_boundary_checked`、`public_boundary` |

| 温度 | 改稿时能不能开口求助 | 通过后第一条消息发给谁、怎么发 | `route_trust`、`graduation_temperature` |

## 共用事件拍点

这四拍为所有路线共享骨架。路线变体只能改变谁在门口、谁赶流程、谁只发消息、补交先后和消息温度，不能改变事件功能。

| 拍点 | 页面 ID | 母版动作 | 玩家功能 |
|---|---|---|---|
| 第1拍：答辩顺序表 | `ACT7-E07-COMMON-B01` | C407 或答辩教室外贴出顺序表，E06 的修改稿版本进入现场。 | 确认本路线答辩前状态。 |
| 第2拍：门口等待 | `ACT7-E07-COMMON-B02` | 门口、走廊、打印店或消息框里出现“等 / 不等”的现实选择。 | 检验关系温度和时间债。 |
| 第3拍：答辩后补交 | `ACT7-E07-COMMON-B03` | 答辩通过或需小修后，签字、小章、扫描、上传和补交顺序压到眼前。 | 把通过结果落成现实手续。 |
| 第4拍：消息回声 | `ACT7-E07-COMMON-B04` | 第一条答辩后消息发出，进入毕业照、饭局和站口温度。 | 进入 `ACT7-E08-GRAD-PHOTO`。 |

## 路线变体页

### `ACT7-DEFAULT` / `POOL-DEFAULT-4XX`

路线功能：默认宿舍修补线中，4XX 可以在答辩门口短暂同场，但等不等不等于亲密恢复。答辩后的补交顺序会继续检验“帮忙”和“兜底”的边界。

核心物件：答辩顺序表、C407 小章、修改后提交页、4XX 群里的“过了”。

| 顺序表 | `ACT7-E07-DEFAULT-P01` | 林亦舟看到自己的顺序，E06 的 `thesis_revision_v2.docx` 被列为答辩用版。 | A. 自己核材料；B. 问 4XX 谁在附近。 | A：`defense_stability +1`；B：`dorm_visibility +1`, `old_debt +1`。 |

- 这条线不是答辩门口大团圆，而是“还会在场，但不再自动亲近”。
- 4XX 的帮助必须短、具体、可拒绝。
