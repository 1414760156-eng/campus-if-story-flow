# 开发长线_IF可玩支线R5-TANG唐骁线

## 用途

本文用于把 `R5-TANG` 从第五幕站队分歧骨架扩展成可拆剧情页、选项页、变量回调和毕业结算的 A 级命运外流线。

它不是小说正文，也不是支线外传。它规定的是：玩家在第五幕强冲突后选择靠近唐骁时，林亦舟的后续大学生活如何进入“规则、流程、项目、责任边界、材料证据和低温实用照看”的命运池。母本公共事件照常发生，但镜头、可操作内容和关系后果都按唐骁线重新嫁接。

## 路线定位

路线编号：`R5-TANG`

路线池编号：`POOL-R5-TANG`

路线等级：A 级核心支线

共享范围：第一卷、第二卷共享；第三幕以后只按关键选择进入对应命运池；本线入口在第五幕站队分歧。

核心体验：林亦舟没有选择留在 4XX 公共桌做平均修补者，也没有选择周屿的热闹、陆沉的沉默边界或摆烂回避，而是先走向唐骁。唐骁会因此更愿意把流程、表格、项目和证据交给他，也会更频繁把林亦舟拖进“先说清楚”“先把责任拆开”“先别靠临时好说话补漏洞”的生活半径里。

本线重点不是“唐骁永远正确”，也不是“项目稳定就等于关系稳定”。唐骁真正的问题是：他相信规则能保护人，却容易在害怕混乱时把规则说成判决；他想防止别人临时推责，却会让周屿觉得被审、让陆沉觉得自己被表格代替。玩家要在一次次流程里决定：是继续把事情做稳，还是让规则回到能让人继续共处的位置。

## 不改变的母本锁点

- 第五幕宿舍强冲突必然发生，且不会因为玩家靠近唐骁就证明唐骁完全正确。
- 周屿、陆沉仍按母本生活半径推进：周屿继续面对玩笑、活动圈、面子和公开误读；陆沉继续面对勤工、图书馆、家庭和发言边界。
- 第六卷的 C407 项目、课程演示、材料归档、关系定形仍发生，只是林亦舟更多从唐骁的流程侧和专业协作侧进入。
- 第七卷的预登记、毕业活动材料、开题 / 答辩、毕业照、清寝、最后一顿饭、离校手续仍发生。
- 5X 仍是唯一硬外流。本线内部回流不能临时打开 5X，也不能跳入周屿线、陆沉线或感情线主分支。

## 核心变量

| 变量 | 含义 | 高值倾向 | 风险 |
|---|---|---|---|
| `tang_trust` | 唐骁是否愿意把关键流程和非功利担心交给林亦舟 | 唐骁愿意解释规则背后的害怕和照看 | 可能把林亦舟当成默认协作者 |
| `project_stability` | 项目、材料、演示和归档是否稳定 | C407、报告、共享文档和答辩材料更稳 | 情绪问题被持续延后 |
| `rule_pressure` | 规则带来的压迫感 | 责任清楚，漏洞减少 | 周屿 / 陆沉觉得自己被表格推远 |
| `responsibility_clarity` | 责任、授权、提交口径是否清晰 | 公开材料风险降低 | 需要林亦舟承担解释成本 |
| `fairness_cost` | 公平执行造成的人情损耗 | 不再靠临时好说话补漏洞 | 饭局、寝室和活动圈低温 |
| `emotion_delay` | 情绪被项目和流程延后的程度 | 任务更稳 | 第七卷更可能只剩体面距离 |
| `public_boundary` | 公开材料、署名、权限和共享范围 | 授权清楚，毕业风险降低 | 流程变慢，唐骁会焦躁 |
| `zhou_misread` | 周屿对林亦舟偏向唐骁的误读 | 周屿减少严肃沟通 | 玩笑更刺、活动圈回声变冷 |
| `luchen_proxy_risk` | 陆沉是否被表格或他人代言 | 低值时陆沉边界更稳 | 高值会让陆沉进一步退开 |
| `dorm_warmth` | 4XX 的日常温度 | 四人还能当面处理 | 低值时只剩专业协作或礼貌同框 |
| `graduation_temperature` | 毕业结算温度 | 继续协作、实用照看、体面距离、低温淡化或不再私聊 | 由前述变量综合决定 |

## 体量与密度

建议 8 章开发体量，对应约 60,000-90,000 字正文容量。本文只提供可开发骨架，不直接写足正文。

最低配置：

- 至少 32 次有效选择。
- 至少 6 个内部回流点。
- 每个内部回流点至少 2 个回流方向。
- 每个回流方向至少 3-4 个连续剧情拍点。
- 至少 3 次强后果回调：第六卷 C407 / 评分细则 / 第七卷归档或离校手续。

## 章节开发详表

### 第一章：时间线不是判决书

章节目标：确定玩家进入唐骁线后的第一种代价。林亦舟靠近唐骁，会让唐骁觉得至少有人愿意先核事实；也会让周屿觉得自己被表格定性，让陆沉担心自己的沉默被写进别人的结论。

现实触发：
- 4XX 事项群只剩短句。
- 唐骁整理出时间线、责任表或材料缺口。
- 周屿抗拒被表格定罪，陆沉不想被人替他说。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |
|---|---|---|---|
| `TANG-CH01-N01` | 读取站队入口 | 4XX 事项群 / 公共桌 | 林亦舟、唐骁、周屿、陆沉 |
| `TANG-CH01-N02` | 决定先找唐骁 | 公共桌 / C407 共享文档 | 林亦舟、唐骁 |
| `TANG-CH01-N03` | 处理第一份时间线 | 共享文档 / 4XX 公共桌 | 林亦舟、唐骁 |
| `TANG-CH01-N04` | 周屿和陆沉回声 | 事项群 / 楼道 / 床位边 | 周屿、陆沉软回声 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `TANG-CH01-C01` | 帮唐骁核事实，但删掉定罪感措辞 | `tang_trust +1`, `public_boundary +1`, `responsibility_clarity +1` | 唐骁接受删改，但追问删完如何说明问题 | 第二章唐骁愿意把共享文档权限给林亦舟 |
| `TANG-CH01-C02` | 支持唐骁按原表发群里 | `project_stability +1`, `rule_pressure +2`, `zhou_misread +2` | 事情推进快，周屿直接用玩笑顶回去 | 第五章周屿更容易把林亦舟归到审判侧 |
| `TANG-CH01-C03` | 要求先问每个人能不能被写进表 | `public_boundary +2`, `emotion_delay +1`, `tang_defense +1` | 唐骁觉得低效，但没有拒绝 | 第七章归档权限更容易处理 |
| `TANG-CH01-C04` | 只私下看表，不让它发群 | `tang_trust +1`, `responsibility_clarity 0`, `dorm_warmth +0.5` | 唐骁暂时压住表，事情没有公开推进 | 第三章公平争论会重新爆出 |

内部回流：调用 `RET-TANG-01` C407 清单。

防漂移：
- 不写成唐骁拿表格审判所有人。
- 不让一次删措辞直接修好宿舍关系。
- 不让周屿或陆沉因为表格短时间内消失。

### 第二章：C407 的版本号

章节目标：把“靠近唐骁”从情绪选择推入现实任务。C407 演示、共享文档版本、评分细则和提交截止会逼玩家判断：先稳项目，还是把被项目压住的情绪拉回现场。

现实触发：
- C407 项目文档出现多个版本。
- 软件工程课程设计需要演示口径。
- 4XX 同时还有没说完的旧账。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |
|---|---|---|---|
| `TANG-CH02-N01` | 收到版本冲突提醒 | C407 / 共享文档 | 林亦舟、唐骁 |
| `TANG-CH02-N02` | 评分细则确认 | C407 / 机房 / 课程群 | 林亦舟、唐骁、老师软回声 |
| `TANG-CH02-N03` | 宿舍旧账催回 | 事项群 / 公共桌 | 周屿、陆沉软回声 |
| `TANG-CH02-N04` | 版本号回流 | 共享盘 / 归档文件夹 | 唐骁、林亦舟 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `TANG-CH02-C01` | 跟唐骁先稳项目，情绪晚点谈 | `project_stability +2`, `emotion_delay +1`, `tang_trust +1` | 唐骁效率明显提高 | 第五章周屿会说“你们俩又开会” |
| `TANG-CH02-C02` | 项目做一半，强行拉回宿舍谈旧账 | `dorm_warmth +1`, `project_stability -1`, `rule_pressure -1` | 唐骁停下，但脸色变硬 | 第七章演示前更容易出现补救压力 |
| `TANG-CH02-C03` | 先把版本权限锁清楚，再发一句约时间 | `responsibility_clarity +2`, `emotion_delay +0.5` | 唐骁认可处理顺序 | 第六章他更可能承认自己怕漏洞 |
| `TANG-CH02-C04` | 让每个人只确认自己的部分，不合并情绪判断 | `public_boundary +1`, `luchen_proxy_risk -1`, `project_stability +1` | 陆沉压力降低，周屿仍觉得冷 | 第三章公平代价更具体 |

内部回流：调用 `RET-TANG-02` 评分细则误读。

防漂移：
- C407 不是天然和解场。
- 项目稳定不能替代关系修补。
- 唐骁线的专业协作必须伴随宿舍温度代价。

### 第三章：公平的代价

章节目标：让唐骁的规则从“推进任务”变成“关系压力”。玩家要处理最低责任、可协商部分、排班、活动和沉默边界，判断公平究竟是保护人还是把人推远。

现实触发：
- 周屿有活动排练，陆沉有勤工或图书馆安排。
- 唐骁要求每个人按规则承担任务。
- 林亦舟知道每个人都有理由，但理由不能把责任全部抹掉。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |
|---|---|---|---|
| `TANG-CH03-N01` | 责任拆分会议 | 公共桌 / C407 | 四人 |
| `TANG-CH03-N02` | 最低责任表 | 共享文档 / 课程群 | 林亦舟、唐骁 |
| `TANG-CH03-N03` | 周屿活动撞期回声 | 活动群 / B206 门口 | 周屿软照面 |
| `TANG-CH03-N04` | 陆沉不被代言 | 图书馆 / 事项群 | 陆沉软回声 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `TANG-CH03-C01` | 支持唐骁按规则执行 | `rule_pressure +2`, `tang_trust +1`, `dorm_warmth -1` | 事情推进最快 | 周屿和陆沉都减少解释欲 |
| `TANG-CH03-C02` | 帮唐骁拆成最低责任和可协商部分 | `responsibility_clarity +2`, `project_stability +1`, `fairness_cost +1` | 唐骁不太满意，但能接受 | 第六章唐骁能说出“不是所有事都一样重” |
| `TANG-CH03-C03` | 让唐骁自己去说，不替他当翻译 | `tang_growth +1`, `conflict_heat +1`, `tang_trust 0` | 现场变硬，唐骁第一次被迫换说法 | 第八章实用照看更可能带一点私人温度 |
| `TANG-CH03-C04` | 先问每个人今晚最低能交什么 | `luchen_proxy_risk -1`, `zhou_misread -1`, `project_stability +0.5` | 规则变慢，但不再像审判 | 第五章误读强度降低 |

内部回流：调用 `RET-TANG-04` 周屿活动设备撞期。

防漂移：
- 公平不是万能正确答案。
- 体谅也不能变成无限兜底。
- 陆沉的压力不能由林亦舟或唐骁替他说完整。

### 第四章：打印店终版

章节目标：处理唐骁线里的实用照看：纸质备份、打印格式、错页、钱和时间。唐骁的关心不容易说出口，但会藏在“我顺手多打了一份”“这个格式别交错”里。

现实触发：
- 演示材料需要纸质备份。
- 打印店排队，生活费和打印费用成为小压力。
- 沈嘉禾可以看格式，但不该被拖成背锅人。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |
|---|---|---|---|
| `TANG-CH04-N01` | 终版材料确认 | C407 / 共享文档 | 林亦舟、唐骁 |
| `TANG-CH04-N02` | 打印店排队 | 打印店 / 食堂侧门 | 林亦舟、唐骁、沈嘉禾软照面 |
| `TANG-CH04-N03` | 错页或漏签 | 打印店 / 4XX 公共桌 | 林亦舟、唐骁 |
| `TANG-CH04-N04` | 费用和备份回流 | 支付记录 / 文件夹 | 四人软回声 |

关键选择：

| 选择 ID | 选项 | 变量变化 | 即时反馈 | 后续回声 |
|---|---|---|---|---|
| `TANG-CH04-C01` | 按唐骁要求多打一份备份 | `project_stability +1`, `money_pressure +1`, `tang_trust +1` | 唐骁嘴上说必要，实际松一口气 | 第七章归档更稳 |
| `TANG-CH04-C02` | 为省钱只打必须页 | `money_pressure -1`, `project_stability -0.5`, `fairness_cost +1` | 唐骁不高兴，但不能说没有道理 | 第六章他会提“风险是谁承担” |
| `TANG-CH04-C03` | 找沈嘉禾只看格式，不让她承担结论 | `public_boundary +1`, `responsibility_clarity +1` | 沈嘉禾愿意帮一眼，唐骁认可边界 | 第五章外人介入误读降低 |
| `TANG-CH04-C04` | 发现错页后先承认自己没核完 | `tang_trust +2`, `repair_depth +1`, `time_debt +1` | 唐骁没有立刻指责 | 第六章唐骁更可能暴露自己的焦虑 |

内部回流：调用 `RET-TANG-05` 打印店终版、`RET-TANG-03` 沈嘉禾边界协助。

防漂移：
- 打印店不是纯生活气氛，必须承担钱、时间、责任或边界。
- 沈嘉禾只能协助格式和边界，不开启感情线。
- 唐骁的实用照看不能直接等于亲密修复。

### 第五章：被误读成审判

章节目标：让唐骁线承受未选择者后果。周屿会把流程理解成审判，陆沉会担心自己被表格代言，外部活动圈和项目流程会同时要求林亦舟表态。

现实触发：
- 周屿活动设备或主持彩排与 C407 演示撞期。
- 陆沉的提交部分需要确认，但他没有授权别人解释他的原因。
- 唐骁要求先按流程走。

可拆节点：

| 节点 ID | 功能 | 场景 | 人物 |

## 对话逻辑

### 唐骁

- 先讲顺序，再讲责任，最后才可能露出担心。
- 他不是喜欢控制别人，而是害怕所有人都用“差不多”“晚点补”“你先帮一下”把漏洞推走。
- 他表达在意通常不直接说“我担心你”，更可能说“这个你别漏”“我多打了一份”“你那栏我没动”。
- 如果玩家长期支持原表发出，他会更依赖林亦舟的流程协作；如果玩家长期当众否定规则，他会觉得林亦舟也在把问题推回混乱里。

### 林亦舟

- 不能变成唐骁的表格执行器。
- 玩家每次靠近唐骁，都要承担“周屿觉得被审、陆沉担心被代言”的代价。
- 他可以帮唐骁把事情做稳，但必须选择是否让规则保留人的温度。

### 周屿

- 在本线里不抢主轴，但会通过玩笑、活动撞期、少解释和不再主动拉林亦舟去热闹里形成误读回声。
- 他不是因为讨厌唐骁，而是觉得林亦舟把“事实清楚”放在“别让人当众难看”之前。

### 陆沉

- 在本线里更容易担心自己被表格或别人代言。
- 陆沉的回声要通过“只确认自己的部分”“不补充解释”“把未授权内容删掉”表现。

### 沈嘉禾

- 沈嘉禾是资料和边界照面，不是本线攻略对象。
- 她可以帮忙看格式、提醒授权、指出引用范围，但不会替 4XX 判断谁该承担关系责任。

### 秦越 / 许棠

- 秦越只作为活动圈与流程撞期压力出现。
- 许棠可以在公开材料、照片、活动短稿里提醒授权，但不能替唐骁线完成关系修复。

## 漂移防线

- 本线不是“唐骁正确线”。靠近唐骁会增加项目稳定和责任清晰，也会增加规则压迫、情绪延后和未选择者误读。
- 本线不是“完美线”。完美线写公开信用和被工具化；唐骁线写站队后规则、流程、责任边界和实用照看。
- 本线不是“学习项目线”。C407 和材料只是现实载体，真正可玩的是规则如何保护人又伤人。
- 本线不能把周屿写成不讲理反派，也不能把陆沉写成被表格覆盖的背景。
- 本线不能让内部回流打开其他命运池。所有回流只改变本线关系温度、误读、资源代价和毕业结算。
- 本线不能取消母本公共锁点，只能改变林亦舟进入锁点的位置。
