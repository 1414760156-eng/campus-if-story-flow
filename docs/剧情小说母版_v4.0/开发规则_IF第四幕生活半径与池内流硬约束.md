# IF 第四幕生活半径与池内流硬约束

更新时间：2026-05-24

## 用途

本文用于规范第四幕的可玩剧情密度、选择点数量和池内内流方向，避免第四幕只服务工作线，或早期外流线从第三 / 四幕直接跳到第六幕。

核心规则：

- 第四幕是全路线生活半径分化层。
- 工作线 `R4-WORK` 的第四幕是主场。
- 其它路线即使第五幕才正式锁定，也必须在第四幕留下生活半径、变量前置和关系温度差异。
- 第四幕不是第六幕收束卷，不能套第六幕双选。第四幕选择窗至少保持三项：2 个池内微内流点 + 1 个当前支线主轴推进。

## 最低选择剧情点

这里的“选择剧情点”不是按钮数，而是一组完整小剧情：

```text
选择前铺垫场景
-> choice_window
-> 选择后反馈场景（每个方向各自拥有）
-> 选择后承接剧情（继续走当前路线或进入微内流小链）
-> 至少 1 个变量变化
-> 至少 1 个 act5_echo_hook
```

一个场景里放 3-4 个按钮，不等于 3-4 个选择剧情点。能计数的必须是会改变第四幕生活半径、第五幕入口条件或第六幕回响的剧情节点。

选择窗不是菜单页。任何可计数选择剧情点都必须先有剧情铺垫，再让玩家选择；选择后不能只显示变量，也不能直接跳到下一个选择。每一个方向都要有本方向独立反馈：

- 当前支线主轴推进方向：必须有选择后行动段，说明林亦舟实际做了什么、谁看见、谁误读、下一段剧情如何继续。
- 池内微内流方向：必须进入 3-5 段小剧情链，至少包含触发、对话 / 行动、小抉择、回声。
- 低风险或沉默类方向：也必须有反馈，用缺席、未回复、物件位置、群消息变短、别人少问一句等方式表现，不允许空跳。

## 第四幕叙事职责

第四幕不能被写成“从第三幕到第五幕的过场”。它至少承担四个叙事职责：

1. **拉开时间感**：让玩家感到暑假、返校前、校内外生活半径确实经过了一段时间，而不是点完第三幕马上进入第五幕。
2. **确认生活半径**：用具体地点、任务、消息和同场人物，证明林亦舟正在靠近某种生活方式。
3. **积累现实压力**：让钱、时间、活动、项目、家庭、公开材料、缺席或静音变成可读代价。
4. **递交第五幕钩子**：第四幕末尾必须留下第五幕能读取的变量和场景余波，不能只用一句“之后进入第五幕”跳转。

因此，下面的 4 / 5 个选择剧情点只是最低验收线，不是正式写作目标。正式开发时应按“推荐剧情密度”执行。

## 小说母版沉浸密度硬规则

玩家不是在答题，而是在进入小说母版的另一种生活半径。剧情页必须吸收母版中已经写出的场景、人物对话、物件、地点动线和生活噪音；不能只把母版压缩成“事件摘要 + 选项表 + 变量结果”。

### 剧情页不是问答页

禁止把可玩剧情写成以下节奏：

```text
一句背景
-> 三个选项
-> 一句结果
-> 下一个选项
```

正式剧情页必须保持小说化承载：

- 每个选择剧情点前，应至少有 2-4 个连续剧情页或等量剧情段，展示玩家为什么要选。
- 每个选择剧情点后，应至少有 1 个方向反馈页；微内流方向还要继续展开 3-5 段小剧情链。
- 每两个选择剧情点之间，必须有非选择剧情推进；不能连续出现“选择 -> 反馈一句 -> 选择”。
- 选择频率不能压过叙事密度。选项是剧情中的动作，不是把剧情切碎的题目。

### 母版内容进入剧情页的最低要求

每个母本锁点改写成 IF 剧情页时，至少要保留并改写以下内容中的 4 类：

| 内容类型 | 要求 | 示例 |
|---|---|---|
| 场景地点 | 使用母版真实地点和动线，不写抽象空间 | 青枫居、C201、B204、图书馆、打印店、快递站、武生院站 |
| 人物对话 | 至少保留或改写 2 轮人物间具体话语逻辑 | 唐骁提醒资料包、周屿玩笑被截住、陆沉说“不用替他”、母亲问联系人 |
| 物件细节 | 用物件承载压力，不只说明变量 | 学生证、黑色签字笔、饭卡、打印材料、转账备注、四栏纸、绿豆糕 |
| 现实流程 | 保留母版流程的具体步骤 | 排队、签字、打印、查成绩、补考、填联系人、班会核对 |
| 关系余波 | 写出他人如何看见、少问、误读或等待 | 4XX 少叫一次饭、晚风只要报平安、辅导员不接受代签 |

如果一个锁点只剩“功能说明”和“变量名”，不计为正式剧情页，只能算开发索引。

### 母版锚点扩写规则

母版不是限制剧情页只能逐字搬运原文。若某条路线需要更多沉浸页，而母版没有逐段写出对应小事件，可以基于母版锚点扩写小剧情，但扩写必须贴着母版生活逻辑走。

因为 IF 线数量远多于母版本文，不是所有支线小事都会被母版逐字记录。正式开发不要求每个剧情页都能在母版里找到一段一模一样的原文；要求是每个新增页都能回答清楚：它从哪个母版地点、人物关系、现实流程、物件或变量压力生长出来，为什么仍属于当前路线池。

允许扩写：

- 在母版已有地点内补小动作：打印店排队、C201 门口等人、B204 核对名单、图书馆登记台找表。
- 在母版已有人物关系内补对话：唐骁提醒资料、周屿玩笑收住、陆沉只给流程、晚风要求报平安、父母围绕联系人拉扯。
- 在母版已有现实流程内补阻滞：资料没印全、饭卡余额不够、窗口排队、学生证忘带、表格备注写不下。
- 在母版已有物件上补回声：黑色签字笔、学生证、转账备注、绿豆糕、四栏纸、门禁卡、补考资料。

禁止扩写：

- 新增会改变母版事件因果的大事件。
- 新增抢走当前路线主轴的新人物事件。
- 把软照面扩写成新完整路线入口。
- 为了加页数写脱离地点、人物、流程和变量的散文段。

判断标准：删掉扩写段后，母版公共锁点仍能成立；保留扩写段后，玩家只会更清楚当前路线生活半径，不会误以为进入了另一条路线。

### 单页正文密度建议

正式 JSON / 试玩层中的剧情页应按正文页处理，不按表格摘要处理：

- 普通剧情页：建议 180-350 字，至少包含地点、动作、人物反应中的两项。
- 关键冲突页：建议 350-700 字，必须包含对话、动作、物件或环境压力。
- 选择前铺垫：不低于一个完整场景节拍，让玩家知道这不是抽象价值题。
- 选择后反馈：不低于一个完整动作或对话节拍，让玩家看见选择怎样改变现场。

字数不是唯一目标，但低于这些密度时必须能证明页面仍有完整场景、人物和动作。纯说明、纯变量、纯系统提示不算沉浸剧情。

### 缩写不是减页规则

把小说母版改写成 IF 剧情页时，压缩对象是冗余表达、重复解释和不改变关系位置的对白，不是剧情层数、选择前因和选择后承接。正式写回时可以让单页更短，但不能把 3-5 个连续场景压成一个摘要页，也不能把玩家做选择前必须经历的现场压没。

压缩优先级：

- **保留**：会改变变量、关系边界、误读、承诺、拒绝或选择理解的动作、物件、地点和关键对白。
- **转述**：寒暄、重复确认、没有改变站位的玩笑和流程问答，可以改为第三人称交代。
- **删除**：同义反复、已经被动作表达过的心理解释、纯气氛堆叠、对当前选择没有信息增量的对白。

对话保留原则：只有会改变边界、误读、承诺、拒绝或选择含义的句子才逐句写。其它对白应用第三人称概括。例如“周屿先说自己差点点进去，唐骁追问他哪门课需要，周屿把玩笑绕回林亦舟，说只是替他先踩个雷”即可交代群聊背景；真正需要保留的，是林亦舟明确截住边界的“不要替我”。

页数保留原则：正式页可以更紧，但场景数和反馈数不能骤减。一个母本锁点若原本需要“选择前 3-4 页 + 选择窗 + 每方向反馈页”，压缩后仍应保留连续页级节奏，只压每页内容密度，不把它压成“摘要 + 选择 + 结果”。

### 前因后果焦点规则

每一页必须先回答“这一页讲什么因果”。如果这一页讲差成绩，重点就应是查成绩的过程动作、成绩出现的瞬间、最关键人物反应和由此产生的下一步压力；不能写成每个人都轮流反应一下。其它人物可以带过、延后到下一页，或只作为字段备注里的回声来源。

单页焦点建议：

- 查成绩页：登录 / 刷新 / 找到科目 / 截图保存 / 关键现场反应。
- 资料包页：风险链接 / 官方提醒 / “替人踩雷”误区 / 主角截住边界。
- 复习页：坐到具体地点 / 拆资料 / 写第一行 / 白天可处理的现实理解。
- 打印页：文件进入打印机 / 纸张和证件 / C201 路线确认 / 选择前压力成形。

人物反应选择原则：一页最多保留 1 个主反应人物。其它人物只做背景消息、延后回声或一句转述，避免“这个反应一下，那个反应一下”的散点页。

正文 / 备注分离原则：玩家可见正文里不得出现“这一页要表达”“这个方向的结果”“字段备注”“变量增加”等开发记录句。变量、设计意图、写回说明应放在正文后的字段备注表或 JSON 字段里；正文本身只写人物、动作、对话、物件和现场后果。

## 数量硬标准

| 第四幕类型 | 适用路线 | 最低选择剧情点 | 每个选择窗方向数 | 说明 |
|---|---|---:|---:|---|
| 普通生活半径层 | `DEFAULT-4XX`、`R5-WANFENG`、`R5-ROMANCE`、`R5-ZHOU`、`R5-TANG`、`R5-LUCHEN`、`R5-LIEFLAT`、`R5X-HARD` 的第四幕前置 | 4 | 至少 3 | 不一定锁线，但必须记录生活半径和变量前置。 |
| 早期外流池内层 | `A3-ACTIVITY-PUBLIC`、`R3-PERFECT` | 4 | 至少 3 | 第三幕已经进入父池 / 子外流后，第四幕必须在当前池内演化。 |
| 第四幕主场线 | `R4-WORK`，以及未来任何以第四幕为主场的新线 | 5 | 至少 3 | 工作线的第四幕必须达到与后续主线选择密度相称的完整路线体量。 |

最低保证：

- 所有路线第四幕至少 4 个选择剧情点。
- `R4-WORK` 至少 5 个选择剧情点。
- 每个第四幕选择窗至少 3 个方向，标准结构是 2 个池内微内流点 + 1 个当前支线主轴推进。
- 如果第四幕还未正式锁池，三项选择记录多个候选倾向和第五幕入口偏置；一旦写入 `route_pool_id`，三项选择仍必须留在当前池内，不得变成其它完整路线入口。

## 推荐正式密度

为避免玩家感觉第四幕过短，正式剧情页不应只贴着最低线写。

| 第四幕类型 | 最低剧情段 | 推荐剧情段 | 最低选择剧情点 | 推荐选择剧情点 | 过幕要求 |
|---|---:|---:|---:|---:|---|
| 普通生活半径层 | 8 | 10-12 | 4 | 4-5 | 至少 1 个非选择过场场景 + 1 个第五幕入口钩子。 |
| 早期外流池内层 | 10 | 12-14 | 4 | 5 | 至少 2 个当前池专属场景，不能只复用公共主轴。 |
| 第四幕主场线 | 12 | 14-16 | 5 | 6-7 | 至少 3 个路线主场场景，并在末尾明确第五幕承接压力。 |

剧情段定义：

- 一个剧情段应有明确地点、人物动作、事件压力或关系变化。
- 纯系统提示、单句变量说明、单按钮确认页不计入剧情段。
- 每两个选择剧情点之间至少要有 1 个非选择剧情段，避免玩家连续点选项直接滑进第五幕。
- 第四幕最后必须有一个“过幕承接段”，说明玩家带着什么压力进入第五幕。
- 每个方向选择后至少有 1 个反馈段；主轴推进方向同样需要反馈和承接，不能只给变量结果。
- 表格中的“微剧情链 / 结果”只算设计索引，不能替代正式剧情页或 JSON 中的选择后反馈正文。

正式开发建议：

- `A3-ACTIVITY-PUBLIC` / `R3-PERFECT` 不应少于 12 个剧情段、5 个选择剧情点。
- `R4-WORK` 不应少于 14 个剧情段、6 个选择剧情点；最低 12 / 5 只用于极限压缩版本。
- 其它第五幕才锁定的路线，第四幕若只做前置，也建议达到 10 个剧情段、4-5 个选择剧情点。

## 四个必经功能位

所有路线第四幕至少覆盖以下 4 个功能位。不同路线可以改写场景和人物，但不能删除功能。

| 功能位 | 作用 | 可写成的玩家动作 | 必须留下的后果 |
|---|---|---|---|
| `ACT4-RADIUS-01-DAILY-ANCHOR` | 确认日常生活落点 | 回宿舍、去活动、接排班、回消息、保持静音 | `act4_daily_radius`、当前生活半径分数 |
| `ACT4-RADIUS-02-RESOURCE-PRESSURE` | 记录资源压力 | 花时间、花钱、请假、交材料、补证明 | `time_debt`、`money_pressure`、`project_pressure` 等 |
| `ACT4-RADIUS-03-RELATION-ECHO` | 记录关系回声 | 解释、沉默、赴约、错过、只给最低事实 | `dorm_warmth`、候选关系温度、误读或缺席 |
| `ACT4-RADIUS-04-ACT5-HANDOFF` | 把第四幕压力交给第五幕 | 带着什么回校、先找谁、是否公开、是否继续回避 | `act5_entry_bias`、`act5_echo_hook` |

第四幕主场线额外增加第 5 个功能位：

| 功能位 | 作用 | `R4-WORK` 示例 | 必须留下的后果 |
|---|---|---|---|
| `ACT4-RADIUS-05-ROUTE-PRESSURE` | 把主场矛盾推到临界点 | 固定班 / 请假 / 工资 / 家庭暗线 / 错过照面 | `route_pool_id`、`route_lock_pressure`、第五幕承接条件 |

## 三选内流方向

第四幕一旦进入当前池，所有选择窗至少保持三项。它和第六幕不同：第六幕是收束卷，才统一二方向；第四幕是生活半径分化层，必须允许更多微影响进入。

### 三选内流定义

第四幕的三项选择不是三条新路线，而是当前池内的三种处理方式：

| 方向类型 | 作用 | 可以带入的人 | 变量结果 |
|---|---|---|---|
| A：当前支线主轴推进 | 继续把时间、注意力、公开位置或现实资源投给当前池主压力 | 当前路线核心人物、当前社团 / 工作 / 宿舍场景中的直接关系人 | 提高当前池投入、稳定当前生活半径，同时增加时间债、缺席或误读 |
| B：池内微内流点 1 | 不离开当前池，但处理一个未选择者、母版公共事件或边缘人物带来的微压力 | 未选择主线人物、4XX 旧关系、晚风消息、社团候选、工作 NPC、家庭消息、公共任务 | 改变软回声、误读、边界、第五幕入口偏置，不改变 `route_pool_id` |
| C：池内微内流点 2 | 从另一个角度处理当前池的边界、资源或关系回声 | 另一个命名人物、另一个公共锁点、另一个现实任务 | 改变第二组微影响变量，制造玩家差异，不改变 `route_pool_id` |

三选结构的标准句式：

```text
推进本支线主轴
vs
处理微内流点 1
vs
处理微内流点 2
```

例如：

- A3 活动父池：继续把公开材料做清楚 / 回 4XX 留一句解释 / 处理候选女生带来的署名边界。
- 完美线：继续按流程补齐归档 / 拒绝再被默认兜底一次 / 回宿舍解释一次私人延迟。
- 工作线：接下固定班保收入 / 请假保住一次宿舍照面 / 去补一份家庭或证明材料。
- 晚风前置：继续保留私人入口 / 把现实边界说给 4XX 听 / 处理一次晚风以外的公开任务。
- 摆烂前置：继续静音省事 / 回一个最低事实 / 去现场但保持低存在感。

### 微影响规则

微影响是第四幕允许不同人物进入当前池的方式，但它只能制造偏置、回声和代价，不能开启新完整路线。

允许的微影响来源：

| 来源 | 可用方式 | 例子 |
|---|---|---|
| 当前池核心人物 | 推动当前路线主压力 | 周屿催活动、唐骁要材料、陆沉给最低事实、晚风问到站时间 |
| 未选择的主线人物 | 软照面 / 消息 / 公共任务 / 误读 | 唐骁在周屿线里只问流程，周屿在唐骁线里只留玩笑，晚风只发一条语音 |
| A3 候选人物 | 候选来源或公开边界，不直接锁感情线 | 沈嘉禾递材料，夏知微发照片，但只改变候选来源和公开压力 |
| 命名边缘人物 | 生活半径证明，不开主路线 | 秦越、许棠、勤工老师、打印店老板、快递站师傅 |
| 母版公共事件 | 把原事件改成当前池镜头 | 班会、C407、活动彩排、成绩 / 补考、返校、家庭消息 |

微影响只能写入这些结果：

- `soft_echo_routes`
- `micro_influence_source`
- `micro_influence_weight`
- `misread`
- `boundary_checked`
- `dorm_warmth`
- `time_debt`
- `money_pressure`
- `public_boundary`
- `candidate_source_seen`
- `act5_entry_bias`
- `act5_echo_hook`

微影响禁止写入这些结果：

- `route_pool_id`
- `active_route_id`
- `route_lock`
- `primary_bond`
- `hard_outflow`
- `ending_candidate`
- 任何 “unlock_new_full_route” 类字段

### 已外流线和未外流线的差异

第四幕必须区分两种状态。

#### 已外流线

适用：`A3-ACTIVITY-PUBLIC`、`R3-PERFECT`，以及进入 `R4-WORK` 后的工作线。

规则：

- 已经有明确 `route_pool_id`。
- 三项选择必须都留在当前池：1 个本支线主轴推进，2 个池内微内流点。
- 其它人物只能作为微影响进入，不能成为新路线候选。
- 第四幕的主要任务是证明“这条线已经开始自己演化”，并把当前池压力交给第五幕。

推荐三选结构：

| 路线 | 主轴推进 | 微内流点 1 | 微内流点 2 |
|---|---|---|---|
| `A3-ACTIVITY-PUBLIC` | 继续公开活动 / 社团任务 | 处理 4XX 缺席 | 处理候选来源或公开边界 |
| `R3-PERFECT` | 继续低风险自控和公开无错 | 处理私人延迟 | 处理工具化风险或宿舍低温 |
| `R4-WORK` | 接现实排班、钱和证明 | 请假 / 解释保住关系入口 | 处理家庭 / 证明 / 工时边界 |

#### 未外流线

适用：第五幕才正式锁定的 `DEFAULT-4XX`、`R5-WANFENG`、`R5-ROMANCE`、`R5-ZHOU`、`R5-TANG`、`R5-LUCHEN`、`R5-LIEFLAT`、`R5X-HARD` 前置。

规则：

- 第四幕不直接写死 `route_pool_id`，而是记录候选倾向和生活半径偏置。
- 选择可以影响多个候选分数，但不能直接锁完整路线。
- 人物微影响可以更开放，但必须以“第五幕入口偏置”的形式结算。
- 第五幕才根据累积变量进入具体池。

推荐三选结构：

| 路线前置 | 主轴倾向 | 微内流点 1 | 微内流点 2 |
|---|---|---|---|
| 宿舍 / DEFAULT 前置 | 回宿舍处理旧账 | 继续保持体面距离 | 先处理家庭或公共任务 |
| 晚风前置 | 保留私人联系频率 | 回到现实关系说清边界 | 处理宿舍旧账或公开任务 |
| 专注感情前置 | 让 A3 候选持续出现 | 降低候选密度，保留公开边界 | 回 4XX 处理缺席 |
| 周屿前置 | 参与活动圈热闹 | 截住玩笑和人情债 | 回宿舍解释一次缺席 |
| 唐骁前置 | 先稳项目 / 材料 | 放慢流程处理人的边界 | 回应周屿或宿舍误读 |
| 陆沉前置 | 完成现实任务 | 等本人说明，不替他说 | 回 4XX 给最低事实 |
| 摆烂前置 | 继续静音 / 缺席 | 回一个最低事实 | 到现场但保持低存在感 |
| 5X 前置 | 看见旧关系变薄 | 仍不提前离场 | 处理一个旧人同场但不回线 |

未外流线第四幕字段应使用 `route_candidate_scores`、`act5_entry_bias`、`soft_echo_routes`，不要提前写 `active_route_id`。

## 内流点剧情密度

第四幕内流点不能写成 1-2 个过场页，也不能只写“收到消息 / 改变量 / 返回主轴”。每个微内流点都必须是一段小型剧情链，保持母版主轴的小说化推进和玩法密度。

### 单个微内流点最低结构

一个可计数的微内流点至少包含 4 拍：

| 拍数 | 功能 | 内容要求 |
|---:|---|---|
| 1 | 触发 | 由命名人物、公共事件、具体地点或物件触发，不用抽象系统提示。 |
| 2 | 对话 / 行动 | 至少 1 轮人物对话，或一个能体现态度的具体动作。 |
| 3 | 小抉择 | 给玩家一个不改变路线但改变关系温度、边界或资源代价的小选择。 |
| 4 | 回声 | 选择后必须留下变量、物件、消息、缺席、误读或第五幕钩子。 |

正式开发建议：

- 普通微内流点：3-5 个剧情段。
- 关键微内流点：5-7 个剧情段。
- 每个第四幕选择窗里，2 个微内流点不能都写成同一种消息回声；至少一个应落到具体场景和人物对话。
- 每个选择窗之后应有选择后反馈，不能点完选项直接跳到下一个窗口。
- 微内流点的 4 拍是最低剧情链，不是给主轴方向免除反馈的理由；主轴方向也必须有“选择后行动 -> 当前路线继续推进”的承接段。

### 微内流点可包含的次级抉择

微内流点内部允许有轻量次级抉择，但只能影响当前池内变量：

- 语气：直接说 / 留半句。
- 行动：去现场 / 发消息。
- 边界：公开说明 / 私下补一句。
- 资源：花时间 / 保留时间。
- 关系：当面解释 / 只给最低事实。

这些次级抉择不能计作新的完整路线选择，也不能改变 `route_pool_id`。它们的作用是让玩家感到“我确实在这个生活半径里扮演”，而不是被剧情自动送进第五幕。

## 母版写法与玩法等价原则

进入支线后，小说母版主轴的写法和玩法不能降低规格。外流只改变镜头和变量，不改变叙事标准。

必须保持：

- **母版事件逻辑**：班会、活动、C407、宿舍、返校、家庭消息、材料、公开边界等公共锁点仍要发生，只是换成当前池视角。
- **母版小说质感**：用地点、物件、动作、对话和关系余波推进，不用“系统判定 / 变量增加”替代正文。
- **母版玩法密度**：选择前要有铺垫，选择后每个方向都要有反馈，选择之间要有过场和人物行动。
- **母版人物在场感**：没被选择的人也按母版生活半径推进，可以以微影响出现，但不能消失，也不能抢成新路线。
- **母版代价逻辑**：支线不是奖励小剧场。靠近某条线必须带来时间、钱、公开边界、宿舍温度、误读、缺席或未来频率的代价。

禁止：

- 支线进入后变成概要式流水账。
- 用 1-2 个节点跳过第四幕生活半径。
- 只写当前核心人物，完全抹掉母版公共事件和其它人物回声。
- 把微内流点写成“看一眼消息然后返回”。
- 因为外流而降低选择数量、对话密度、反馈长度或现实压力。

允许写法：

```json
{
  "choice_window_id": "ACT4-A3-RADIUS-01",

  "direction_count": 3,
  "directions": [
    {
      "id": "keep_activity_public",
      "label": "继续把公开材料做清楚",
      "direction_type": "main_axis",
      "stays_in_pool": true
    },
    {
      "id": "explain_absence_to_4xx",
      "label": "回 4XX 留一句解释",
      "direction_type": "micro_inflow",

    },
    {
      "id": "check_candidate_signature_boundary",
```

禁止写法：

- 不得写成“去周屿线 / 去唐骁线 / 去陆沉线 / 去晚风线”。
- 不得把三项写成 3 条完整路线；三项必须是“1 个本支线主轴 + 2 个池内微内流点”。
- 不得用“继续 / 返回 / 查看”这类无变量按钮凑选择点。
- 不得让第四幕池内选择直接越过第五幕进入 Act6。
- 不得把未选择者写成可重新攻略对象；只能作为软照面、消息、公共任务或只读回声。

## 校验清单

后续补第四幕时至少检查：

- 每条路线是否至少有 4 个选择剧情点。
- `R4-WORK` 是否至少有 5 个选择剧情点。
- 普通生活半径层是否达到推荐的 10-12 个剧情段，或至少解释为什么只用最低 8 段。
- `A3-ACTIVITY-PUBLIC` / `R3-PERFECT` 是否达到推荐的 12-14 个剧情段和 5 个选择剧情点。
- `R4-WORK` 是否达到推荐的 14-16 个剧情段和 6-7 个选择剧情点。
- 每个选择剧情点是否包含选择前铺垫、选择窗、选择后反馈、变量变化和 `act5_echo_hook`。
- 每两个选择剧情点之间是否至少有 1 个非选择剧情段。
- 第四幕选择窗是否至少 3 个方向。
- 每个三选是否满足“1 个本支线主轴推进 + 2 个池内微内流点”的结构。
- 微影响是否只改变软回声、误读、边界、资源压力和第五幕偏置。
- 已外流线是否没有通过微影响打开新路线。
- 未外流线是否只写候选倾向和第五幕入口偏置，没有提前写死 `active_route_id`。
- 是否没有重新开放其它完整路线。
- 是否没有从第四幕直接跳 Act6。
- 第五幕是否能读取第四幕留下的变量和回声。
