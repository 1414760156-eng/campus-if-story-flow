# IF 第七卷 ACT7-E03 最后一个新年剧情页级细稿

## 用途

本文用于把 `ACT7-E03-LAST-NEW-YEAR` 从第七卷母版正文拆成可开发剧情页、选项页、变量回调和 JSON 事件结构。

本文件不是小说正文，也不是试玩器脚本。它服务于后续正式 JSON 拆分：同一个“最后一个新年”事件点，所有路线都必须经过，但每条路线读取自己的 `route_pool_id`、`act7_variant_id` 和 `graduation_temperature`，输出不同镜头、不同文本和不同结局温度。

## 接入位置

| 字段 | 建议值 |
|---|---|
| `event_id` | `ACT7-E03-LAST-NEW-YEAR` |
| `act` | `act7` |
| `mother_chapter` | 第七卷第三章《最后一个新年》 |
| `previous_event` | `ACT7-E02-PUBLIC-SCOPE` |
| `next_event` | `ACT7-E04-THESIS-TOPIC` |
| `route_switch_allowed` | `false` |
| `same_mother_event` | `true` |
| `main_location` | 教学楼 B204 |
| `base_scene` | “最后一个新年”主题材料会、现场核验、不强制公开 |

本事件必须承接前一事件“不公开也成立”，并把压力推进到下一事件“开题不能继续一起空着”。它不能写成毕业晚会、同学会或回忆短片，而要写成物件、来源、公开范围、本人确认和不公开权利的现场核验。

## 共用硬规则

1. 每条路线必须写出“最后一个新年”和“开学处境”的差异。
2. 每条路线必须处理一个具体物件或空白目录，不能只写心情。
3. 每条路线必须让玩家做一次核心二方向选择：公开 / 留存，讲出 / 不讲，带物 / 空白，等本人 / 替说明。二方向按当前路线语境改名，但不能扩成新路线。
4. 每条路线必须写出至少一个后续回声：开题表、毕业照、清寝、最后一顿饭或站口文本。
5. `R5X-HARD` 仍保持 `route_lock = "5x"`，旧线只允许软照面，不得回原多线。

禁止写法：

- 不允许用“大家都想起了四年”覆盖具体冲突。
- 不允许把旧照片、旧物、旧消息默认写成可公开素材。
- 不允许让某条线因为新年材料会跳去另一条感情线或室友线。
- 不允许把空白目录写成失败；空白可以是边界，也可以是错过，取决于路线。
